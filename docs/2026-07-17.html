When Vision Overrides Language: Evaluating and Mitigating Counterfactual Failures in VLAs

1ロボットは本当に“言葉”を聞いているのか？

想像してみてほしい。家庭用ロボットに「赤いコップを取って」と頼んだのに、いつも目の前に置いてある青いコップばかり掴んでしまう。まるで指示を聞いていないかのようなこの現象が、いま最先端のVLAモデルVision-Language-Action: 視覚と言語からロボットの行動を出力するモデルで頻繁に起きている——と、今日紹介する論文は指摘する。ロボットは言葉ではなく、“よく見る景色”に引きずられて動いている疑いがあるのだ。

2背景：VLAは“見たものを繰り返すクセ”を持っている

VLAモデルは、カメラ画像と自然言語の指示を入力として受け取り、ロボットの関節や手先の動きを出力する。学習には主に行動クローニング人間の実演データを真似するように方策を学習させる手法が使われる。大量の実演データからパターンを学ぶうちに、モデルは「この見た目の場面では、だいたいこう動けばよい」という強い経験則を身につけてしまう。

問題は、既存のベンチマークの多くが「タスクに成功したかどうか」だけを見ている点にある。仮にモデルが指示文をまったく無視し、目に映る物体をいつも同じ順番で操作していたとしても、その物体が正解であれば「成功」と記録されてしまう。つまり、モデルが本当に言葉に従っているのか、それとも視覚的な癖に従っているだけなのかを、従来の評価方法では区別できていなかった。

3LIBERO-CF：“指示だけをすり替える”テストで化けの皮を剥がす

この論文はまず、LIBEROロボット操作タスクを模擬するシミュレーションベンチマークの環境をベースに、LIBERO-CFという新しい評価セットを作った。発想はシンプルで、カメラに映る場面(レイアウト)はそのままに、与える指示文だけを差し替える。これは一種の反実仮想「もしこの条件が違ったら」を試すことで、本当の原因を切り分ける考え方的なテストで、もしモデルが視覚だけに頼っているなら、指示を変えても同じ行動を繰り返すはずだ。実際に評価してみると、最先端のVLAの多くがこの“指示すり替え”に弱く、頻繁に元の(視覚的に見慣れた)行動を繰り返してしまうことが明らかになった。

Counterfactual Action Guidance（CAG）：言語の効き目を後から強める

この弱点に対して論文が提案するのが CAG(Counterfactual Action Guidance) という推論時の工夫だ。考え方は二段構えになっている。ひとつは普段どおり画像と言語の両方を条件にした通常のVLA方策。もうひとつは、あえて言語情報を外し、画像だけを条件にした VA(Vision-Action) モジュールだ。この二つの出力を比較することで、「言語を入れたことでどれだけ行動が変わったか」という差分を取り出せる。あとはその差分を強調する方向に最終的な行動を補正すればよい。

この発想は、画像生成モデルで使われるclassifier-free guidance条件付きの出力と無条件の出力の差を利用し、狙った条件の効き目を強める生成モデルの技法によく似ている。論文のアイデアを直感的な擬似コードで表すと、次のようなイメージになる（実装の詳細を簡略化した一例）。

# ① 通常どおり、画像+言語を条件にした行動
1a_lang = VLA_policy(image, instruction, state)①
# ② あえて言語を使わない、画像だけの行動
2a_vision = VA_module(image, state)②
# ③ 「言語のおかげで変わった分」だけを取り出して強調
3guidance = a_lang - a_vision
4a_final = a_lang + lambda * guidance③
CAGの推論イメージ（論文のアイデアを簡略化した例）

この方法の良いところは、追加の学習データも、モデルの改造も要らない点にある。既存のVLAモデルをそのまま使い、推論のやり方だけを変える“プラグアンドプレイ”の解法だ。

シミュレーション上のLIBERO-CFでは、追加学習なしでπ0.5という代表的なVLAに適用するだけで言語追従精度が9.7ポイント、タスク成功率も3.6ポイント向上した。VAモデルと組み合わせるとさらに伸び、言語追従精度は15.5ポイント、タスク成功率は8.5ポイントの改善が見られている。シミュレーションだけでなく実機での評価でも、指示を無視してしまう反実仮想的な失敗が9.4ポイント減り、タスク成功率は平均17.2ポイント向上したという。

VLAは“見たものを繰り返す”クセを持っている。CAGは、そのクセを追加学習なしで矯正する。

5なぜこの研究が重要なのか

この論文の面白さは、派手な新モデルを作ったことではなく、「モデルが本当に指示された条件を使っているか」を測る評価方法そのものを設計し直した点にある。見た目の成功率だけを追いかけていると、モデルの本当の弱点——ここでは言語を無視して視覚のショートカットに頼るクセ——を見逃してしまう。良い評価軸を立てることは、新しい手法を作ることと同じくらい価値がある、という姿勢は研究全般に通じる教訓だ。

6意義と限界

限界 1

評価の中心はLIBERO-CFというシミュレーション環境であり、実機実験は補足的な検証にとどまる。

限界 2

単体のVLAだけでもある程度の改善はあるが、より大きな効果を得るには追加のVAモジュールを別途用意する必要がある。

限界 3

推論時に2つの分岐（言語ありとなし）を計算する必要があり、単純なVLA単体推論より計算コストが増える。

限界 4

根本的にモデルの内部表現の偏りを直すわけではなく、あくまで推論時の“後付けの補正”である点には注意が必要。

💡 R&Dでよく出る考え方
見た目の性能指標が高くても、モデルが「正しい理由」でその答えに辿り着いているとは限らない。この論文のように、条件を意図的にすり替えて反応を見る反実仮想的なテストは、言語処理や画像認識など他分野でもモデルの本当の理解力を確かめる定番の手口になっている。
