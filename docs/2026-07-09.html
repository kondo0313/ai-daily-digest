Feeling the Unexpected: ResTacVLA for Contact-Rich Manipulation via Residual Tactile Representation
Pengwei Zhang, Bin Xie, Ce Hao, Xinpan Meng, Xinyu Guo, Fang Deng, Long Cheng, Tiancai Wang（所属は論文中に明記なし） arXiv cs.RO ・ 2026年7月 原文 →

1触覚は「サプライズ」として使うと強くなる

ロボットアームが物をつかむとき、私たちは視覚だけでなく、指先の感触で「ちゃんと掴めているか」「滑っていないか」を無意識に確かめています。近年注目されているVLAモデルVision-Language-Action: 視覚と言語からロボットの行動を出力するモデルにも、触覚センサーの情報を組み込もうという研究が進んでいます。ところが単純に視覚と触覚を混ぜて学習させると、情報量の多い視覚ばかりにモデルが頼ってしまい、せっかくの触覚情報がほとんど使われなくなる問題が起きます。今日紹介する論文『Feeling the Unexpected: ResTacVLA』は、この問題を「触覚を生データとしてではなく、視覚の予測に対する“意外性”として扱う」という発想で解決しようとしています。

2背景：なぜ触覚は視覚に負けてしまうのか

この現象はモダリティ崩壊複数の感覚情報を同時に使うモデルで、情報量の多い一部の感覚(多くは視覚)ばかりに頼ってしまい、他の感覚が学習に活かされなくなる現象と呼ばれます。柔らかい物をつぶさないように掴む、狭い隙間に部品を差し込むといった接触の多い作業では、視覚だけでは判断できない微妙な力加減が欠かせません。しかし触覚センサーの信号は視覚に比べて次元が小さく、情報としての「密度」も低いため、学習の過程でモデルから軽視されがちです。ResTacVLAは、この情報量の非対称性そのものを設計でひっくり返そうとしています。

触覚を「生データ」ではなく「視覚の予測が外れた分」として扱うことで、疎な信号を密で価値の高い情報に変える。

3手法の全体像：予測符号化からのヒント

着想の元になっているのは予測符号化脳が「予測できる情報」は無視し、「予測から外れた意外な情報」を優先的に処理するという神経科学の理論です。私たちの脳も、慣れた感覚(服が肌に触れている感触など)はほとんど意識しない一方で、「思っていたのと違う」感覚には強く反応します。ResTacVLAはこの仕組みをロボットに応用し、大きく3つの要素で構成されています。

# 視覚特徴から「こう触れたらこう感じるはず」を予測する
1predicted_tactile = predict_from_vision(visual_feat)
# ① 実測値との差分＝サプライズだけを取り出す
2residual = tactile_raw - predicted_tactile①
# ② 差分を有限個の「接触パターン辞書」に離散化する
3contact_token = vq_quantize(residual, codebook)②
# ③ 視覚予測の自信が低いときほど触覚を強く信じる
4gate = sigmoid(-confidence(visual_feat))
5fused_feat = visual_feat + gate * contact_token③
# 融合した特徴からVLAが行動を生成する
6action = vla_policy(fused_feat, language_instruction)

① 残差触覚表現：予測とのズレだけを見る

まず視覚特徴から「こう触れたらこんな感触のはず」という予測値を作ります。そして実際のセンサー値との残差予測値と実際の観測値との差。予測できなかった分の「サプライズ」の量を表すを計算します。滑らず想定通りに掴めている場面では残差はほぼゼロになり、逆に物が滑った、想定外の硬さだったといった場面で残差が大きくなります。これにより、情報量の少ない触覚信号を、モデルにとって「本当に注目すべき瞬間」だけに絞り込むことができます。

② VQボトルネック：接触の「辞書」を作る

取り出した残差はそのままでは連続的でノイズも多い値です。ResTacVLAはこれをベクトル量子化連続的な特徴量を、あらかじめ用意した有限個の代表ベクトル(コードブック)に置き換えて離散的な記号として扱う手法によって、有限個の「接触パターンの辞書」(論文では Latent Contact Primitives と呼ばれる)に変換します。たとえるなら、連続的な音の波形を「あ」「い」「う」といった有限の音素に変換するようなものです。これにより、視覚では捉えきれない「滑った」「弾んだ」「硬いものに当たった」といった離散的なイベントを、モデルが扱いやすい記号として保持できます。

③ 適応的ゲーティング：自信のない時だけ触覚を信じる

最後の工夫が、視覚予測の不確実性モデルが自分の予測にどれくらい自信を持てているかを示す指標。値が大きいほど「自信がない」ことを意味するに応じて、触覚情報をどれだけ混ぜるかを動的に変える仕組みです。視覚がはっきり見えていて自信を持って予測できている場面では触覚をあまり信じず、手元が影になっていたり対象物が視界から隠れていたりして視覚の自信が低い場面ほど、触覚由来の情報を強く反映させます。脳が不意打ちの感覚に強く反応するのと同じ発想を、数式的なゲート機構として実装しているわけです。

4実験でわかったこと

論文では、接触を伴う多様な操作タスクの集合において、ResTacVLAが比較対象となる既存手法をすべて上回ったことが報告されています。特に、物の位置がずれる、予期しない力が加わるといった動的な外乱が起きた場面でも性能が大きく崩れずに済んだ点が強調されています。公開されているアブストラクトの範囲では具体的な成功率などの数値までは明記されていませんが、「視覚だけでは対応しづらい状況でこそ触覚の効果が出る」という論文の主張とも整合する結果と言えます。

5なぜこの発想が面白いのか

「情報量が多いモダリティに小さいモダリティが埋もれてしまう」というモダリティ崩壊は、ロボット工学に限らずマルチモーダル学習全般でよく起きる問題です。ResTacVLAが面白いのは、モデルの構造を複雑にして無理やりバランスを取るのではなく、「そもそも入力として与える情報の定義そのものを変える」という発想で問題を回避している点です。生の触覚値ではなく「視覚が外した分」を入力にすることで、モダリティ間の情報量の差を設計段階で解消しています。これは触覚と視覚に限らず、他のセンサー融合問題にも応用できる考え方かもしれません。

弱いモダリティを「強いモダリティの予測に対する残差」として再定義する——限られたセンサー情報を最大限に活かすための、一般化できる設計原則。

6意義と限界

限界①

アブストラクトの範囲では、具体的な成功率やタスクごとの内訳までは公開されていないため、効果の大きさを正確に見積もるには論文本文の確認が必要です。

限界②

視覚の不確実性をどう推定するかは実装の細部に依存する部分が大きく、推定が不正確だとゲーティングも誤動作しうるという一般的なリスクは残ります。

限界③

扱われているのは主に接触の多い操作タスクであり、非接触タスクや別のロボット形態(ヒューマノイドなど)への一般化は今後の検証課題と考えられます。
