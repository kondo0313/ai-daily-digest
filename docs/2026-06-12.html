1なぜ「二次コスト問題」が現場を悩ませるのか

Transformerは現代のシーケンスモデリング文章・音声・時系列など順序のあるデータを扱う機械学習の分野の王者です。しかしその心臓部である自己注意機構（Self-Attention）には大きな欠点があります。入力の長さを `n` とすると、計算量とメモリが `n²`——つまり二乗に比例して膨らむのです。文が2倍になれば、コストは4倍。長文・長い時系列・大規模コードの処理ではこれが痛烈な制約になります。

そこで登場したのが「サブクワドラティック（二乗未満）アーキテクチャ」。計算量を `n` に対してより緩やかなスケールで抑えながら、Transformerと同等以上の性能を目指す設計群です。ただし、種類が多すぎてどれを選べばよいか分からない……というのが現場の本音でした。この論文はその混乱に正面から向き合います。

2比較対象の3つのアーキテクチャ

論文が取り上げるのは現在最も注目される3アーキテクチャです。それぞれのキャラクターを簡単に紹介しましょう。

xLSTM

古典的なLSTMを大幅に拡張した再帰型モデル。スカラーおよび行列メモリセルに「指数ゲーティング」を導入し、長距離依存を安定して記憶・修正できる設計になっている。

Mamba-2

State Space Model (SSM)線形常微分方程式で系列を圧縮して扱う手法。連続信号処理の理論に基づくの改良版。入力依存の選択的な状態更新が特徴で、言語・音声など幅広いドメインで実績がある。

Gated DeltaNet

線形注意Softmaxの代わりにカーネル関数でAttentionを近似し計算量をO(n)に抑える手法にデルタ則（誤差修正型の記憶書き換え）とゲーティングを組み合わせたモデル。連想記憶的な動作が期待される。

3どんな課題で比べたのか

「単純なベンチマークで勝っても意味がない」という問題意識から、論文は複雑な依存関係を要求する3つのタスクを設定しています。

タスク①

コードモデルの事前学習（code-model pre-training）：プログラムコードは変数スコープ・関数呼び出しなど長距離の構造的依存が密なため、状態追跡能力の試金石になる。

タスク②

大規模LLMからのコードモデル蒸留（distillation）：知識蒸留大きなモデルの出力分布を教師信号に使い、小さなモデルを訓練する手法で、教師モデルの豊かな知識を引き継げるかを検証する。

タスク③

時系列基盤モデルの事前学習（time-series foundation models）：センサーデータや金融時系列など、長い数値系列の傾向・周期を捉えられるかを問う。

5なぜxLSTMが強いのか：統一的な定式化からの分析

論文のもう一つの貢献は、3アーキテクチャを同一の数式フレームで書き直す「統一定式化」を提示したことです。これによって「どの部分が性能差を生んでいるか」を公平に比較できるようになりました。

注目ポイント①：状態追跡（State Tracking）

「状態追跡」とは、過去の情報を圧縮した内部記憶（状態ベクトル）を正しく保持・更新し続ける能力のことです。コードのスコープや時系列の傾向を正しく捉えるには、遠く離れたトークンの影響が今の状態にきちんと反映されていなければなりません。xLSTMは指数ゲーティングにより、この記憶の「積み重ね」と「保持」を柔軟にコントロールできます。

注目ポイント②：メモリ修正のダイナミクス

ゲーティング機構入力に応じてどの情報を記憶・忘却するかを制御する乗算的なスイッチは、古くからLSTMの強みでした。xLSTMはこれをさらに洗練させ、柔軟かつ安定したメモリ修正を実現します。論文の言葉を借りれば「more flexible and stable memory correction via its gating scheme」——ゲーティングの設計こそがxLSTMの優位性の根拠だと結論付けています。一方でMamba-2やGated DeltaNetのゲーティングは構造がやや異なり、複雑タスクでは修正精度が落ちやすいとされています。

💡 R&Dでよく出る考え方：ゲーティングはなぜ大事か
ゲーティングは「水道の蛇口」のようなものです。情報の流量を入力に応じて動的に調節することで、重要な情報は強く通し、不要な情報は絞ることができます。LSTM・GRU・Transformerのすべてに何らかのゲーティングが存在しますが、その設計の細部が長距離タスクの性能を大きく左右します。

注目ポイント③：制御実験による検証

「実際のタスクだけでは条件が複雑すぎて原因が分からない」という批判を先回りするように、論文は合成データによる長さ汎化タスクも実施しています。訓練より長いシーケンスを推論時に与えて、各モデルの状態追跡能力を純粋に測定する実験です。ここでもxLSTMが安定した性能を示し、実タスクの結果が設計原理によるものだという主張を裏付けています。
