1なぜ「長い文脈」を読ませるとAIは遅くなるのか

新人研究者としてTransformerを学び始めると、必ずどこかで「Attention入力の各部分同士の関連度を計算し、重要な部分に注目する仕組みの計算量はトークン数の2乗に比例する」という話にぶつかります。文章が2倍長くなれば、計算量はおよそ4倍になる——これは長い技術文書やコードリポジトリ全体をまとめて読ませたいときに、深刻なボトルネックになります。

たとえば1つのコードリポジトリ全体や、複数の論文をまとめて読ませたい場合、系列長は数万トークンに達します。愚直なAttentionでは、すべてのトークンの組み合わせについて関連度を計算するため、文脈が長くなるほど計算コストが急激に膨らみ、事前学習にかかる時間や計算資源を圧迫します。

今日取り上げる論文は、この「二乗コストの壁」に対して、意外な場所からヒントを持ってきました。天体シミュレーションの世界で古くから使われてきたFast Multipole法という古典的なアルゴリズムです。

2発想の源泉——遠くの星は「まとめて」扱えばいい

何百万個もの星や粒子が互いの重力で影響し合うシミュレーションを愚直に計算すると、やはり粒子数の2乗のペア計算が必要になります。そこで物理学者たちは、こんな工夫をしてきました。「近くの粒子は個別に厳密計算するが、遠くにある粒子の集団は、その代表的な影響力だけを使ってひとまとめの塊として扱う」——これが多重極展開遠くにある物体群の影響を、個々にではなく代表的な集約値で近似的に表す数学的手法の考え方です。

この論文の著者たちは、同じ発想をAttentionに応用できないかと考えました。Attentionにおける「クエリ」と「キー」の関係を、星どうしの重力関係のように捉え直したのです。

3MuSeの仕組み——クエリとキーを意味空間でクラスタリングする

提案手法Multipole Semantic Attention (MuSe) は、クエリとキーをそれぞれ意味的な表現空間でクラスタリングします。あるクエリから見て「近い」クラスタのキーとは通常どおり厳密にAttentionを計算し、「遠い」クラスタのキーに対しては、そのクラスタの要約表現(代表点)とだけやり取りする、という近似を行います。ポイントは、この要約がクエリごとに作られるクエリ固有の要約であることで、キーを固定的にグループ化するだけの方法より精度が高くなります。

# ステップ①: クエリとキーを意味空間でそれぞれクラスタリング
1query_clusters = cluster(queries, k=K)
2key_clusters = cluster(keys, k=K)
# ステップ②: キーの各クラスタについて要約(代表点)を作る
3key_summaries = [summarize(c) for c in key_clusters]①
# ステップ③: クエリごとに「近くは厳密」「遠くは要約」で計算
4for q in queries:
5near_keys = same_cluster_keys(q)
6out_near = softmax_attention(q, near_keys)
7out_far = softmax_attention(q, key_summaries)②
8output[q] = combine(out_near, out_far)

①でクラスタごとの要約を事前に作っておき、②で各クエリが「遠いクラスタ」とはこの要約だけを使って近似的にやり取りします。近いクラスタとは通常どおり厳密に計算するので、精度を大きく落とさずに計算量を減らせる、という仕組みです。

なぜ「文章中の位置」でブロック分けする方法より優れているのか

既存のsparse attention手法の多くは、文章中の位置(たとえば1000トークンごと)でブロックを作る空間的ブロック分割という方法を使ってきました。しかし意味的に関連する内容が、文章中で離れた位置に出てくることはよくあります。たとえばコードの冒頭で定義した関数が、ファイルの終わり近くで何度も呼び出される場合、その関連性は「位置」ではなく「意味」の近さです。MuSeは意味空間でクラスタリングするため、位置に関係なく関連するトークン同士をまとめて扱えます。論文では、同じ計算量(スパース度)で比較したときにこの方式が空間的ブロック分割を上回ったと報告されています。

学習済みモデルにも「後付け」できるという実用性

もう一つの重要な特徴は、MuSeがモデルのアーキテクチャを変更しない近似手法である点です。そのため、すでに学習済みのモデルに対しても、再学習なしにそのまま適用できます。著者らはLlama 3.1-8BとLlama 3.2-1Bという既存の公開モデルに対して、追加学習なしで適用できることを確認しています。新しいモデルを一から作り直さなくても既存の資産に恩恵をもたらせるというのは、研究の実用性を考えるうえで重要な視点です。

位置ではなく「意味の近さ」でAttentionを圧縮する——それがMuSeの核心的な発想。

4意義と、まだわからないこと

この研究の意義は、Attentionの効率化という長年のテーマに対して、機械学習の外側にある古典的なアルゴリズムの知見を持ち込んだ点にあります。Fast Multipole法は数十年前から天体物理やシミュレーション分野で磨かれてきた手法であり、それを「意味空間のクラスタリング」という形でAttentionに翻訳し直したアイデアそのものに新規性があります。
