1視覚の死角 — 接触という「見えない世界」

ロボットがネジを締める、精密な部品を差し込む、ビンの蓋を探りながら回す。こうした作業は人間なら目をつぶっていても指先の感触で補正できます。ところが、カメラ映像だけに頼るロボットにとってこれらは難関です。「ズレている」「滑っている」「詰まっている」という情報が、RGB映像にはほとんど写らないからです。

接触リッチなタスク挿入・組み立て・探索など、物体同士が触れ合うことで初めて進む精密作業において、視覚には根本的な限界があります。コネクタをPCIeスロットに挿すときの「カチッ」という抵抗感、組み立て中に生じる微小なアライメントエラー——これらは目ではなく指先が教えてくれる情報です。では、ロボットに触覚センサーを持たせてAIに統合すれば解決するのでしょうか？ この論文はその「当然の解」に潜む罠を明らかにし、エレガントな突破口を示しています。

2World Action Model とは何か

この論文が拡張するWAMWorld Action Model: 未来の映像（世界状態）を予測しながら、同時にロボットの行動を出力するモデルは、近年のロボット学習研究で急速に注目されているフレームワークです。WAMの基本的なアイデアはシンプルです——「今の状態からAIが行動を生成し、その行動をとったあと世界がどう変わるかを同時に予測する」。

これはロボットが頭の中で未来をシミュレーションしながら行動を決めるようなイメージです。映像を「予測できる」ということは、モデルが世界の物理的ダイナミクスを内部に持っていることを意味します。行動を引き起こした結果まで予測できる分、単なる模倣学習より頑健な意思決定が期待できます。しかしここに、視覚だけでは埋めきれない隙間があります——視覚的に自然な未来でも、物理的に正しいとは限らないのです。

3触覚汚染 — ナイーブな解法が生む新たな問題

「触覚センサーのデータもトークンとしてWAMに入力すれば解決するはず」と誰もが考えるでしょう。ところがこの論文は、それが触覚汚染（tactile pollution）という失敗モードを引き起こすことを初めて体系的に示しました。

触覚信号は映像信号と性質が根本的に異なります。映像は空間的に密（dense）で時間的に滑らか（smooth）に変化しますが、触覚信号は疎（sparse）・局所的（local）・イベント駆動（event-driven）という三つの特性を持ちます。接触していない間はゼロに近く、接触した瞬間だけ突然大きく変化します。まるで静寂の中に突然鳴り響く警報音のようなものです。

この突然のスパイク信号をそのままWAMに流し込むと、拡散モデルノイズからデータを逆算的に復元するジェネレーティブモデル。WAMの行動・映像生成に使われるが受け取る「映像の流れ」の中に、突発的で局所的な信号が紛れ込みます。映像予測モジュールが「この信号は映像として何を意味するのか」と混乱し、結果として映像予測の精度も行動の精度も両方が低下してしまいます。著者たちはこの現象を「触覚が映像と行動を汚染する」として tactile pollution と命名しました。

触覚信号をそのままWAMに入れると、視覚と行動の両方を汚染する——問題は「何を入れるか」ではなく「どう見せるか」にあった。

4Tactile Asymmetric Attention — 「誰に・何を見せるか」を非対称にする

Tactile-WAM が提案する解法の核心は、触覚情報を映像クエリと行動クエリに対して非対称に扱うという発想です。この機構を TAAM（Tactile Asymmetric Attention Mechanism）と呼びます。TAAMは2つのコンポーネントで構成されます。

① VideoClean マスク — 映像は触覚を「見ない」

Transformer入力系列の各要素が他の要素に「注目（Attention）」して文脈を集めるアーキテクチャの注意機構では、クエリ（Q）がキー（K）とバリュー（V）を参照して情報を集めます。VideoClean マスクは、映像生成クエリが触覚トークンのKV（キー・バリュー）を参照することを禁止します。ただし行動生成クエリは引き続き触覚トークンを参照できます。

つまり「映像を生成するAIの目には触覚データを隠す。行動を決めるAIの目には触覚データを見せる」という役割分離です。これにより、映像予測は視覚の世界だけで完結し、イベント駆動の触覚スパイクに乱されなくなります。

② touch-aware bias — 「変化の瞬間」を行動注意に伝える

さらに、予測された触覚状態の変化（「接触が始まった」「滑りが起きた」など）から touch-aware bias を導出します。このバイアスを行動クエリの注意スコアに加算することで、拡散モデルがノイズ除去するプロセス中に「今まさに接触状態が変わった」という信号が行動生成に優先的に反映されます。

たとえて言えば、「工場の監督員（映像モデル）は工場全体の流れを把握するが、アセンブリの担当者（行動モデル）は手先の圧力センサーを直接確認できる」という分業体制です。

6なぜこれが重要か — 「情報の性質」を尊重した設計

この研究の核心的な貢献は、単に触覚を加えたことではなく、異なる性質を持つ感覚モダリティを統合するときの一般的な設計原則を示した点にあります。触覚はテキスト・映像・音声のどれとも異なる情報の性質を持ちます。そのまま混ぜるのではなく、「どのクエリに・どの情報を・どれだけ見せるか」を意図的に設計することが重要だという教訓です。

このアプローチは触覚に限らず応用が利くかもしれません。温度センサー、振動センサー、IMU（慣性センサー）——ロボットが持ち得るさまざまな物理センサーを基盤モデルに統合する際、同じ「非対称性」の原則が有効になる可能性があります。

「どんな情報でもとりあえず全部入れればよい」ではなく、情報の性質に合った見せ方を設計する——これがマルチモーダルAIの次の課題だ。

複数のモダリティを統合するとき、最も簡単な方法は「全部を同じ空間に投げ込んで、モデルに学ばせる」ことです（early fusion と呼ばれます）。しかし本論文が示すように、モダリティ間の統計的性質の違い（密/疎、連続/イベント駆動）が無視できない場合、明示的な設計が必要になります。「どのモダリティをどのクエリに見せるか」という問いは、マルチモーダルアーキテクチャ設計の重要な論点です。
