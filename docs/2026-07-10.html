1はじめに：「記憶の壁」という根本的な問題

大規模言語モデルの世界では長らくTransformer自己注意機構を使った大規模言語モデルの基本アーキテクチャが主流です。しかしTransformerには致命的な弱点があります。入力が長くなるほど計算量とKVキャッシュ過去トークンのKey・Valueを保存するメモリ。長文では膨大になるのサイズが二乗的に膨れ上がるのです。

そこで注目されてきたのがLinear RNN各ステップの計算量が固定で、状態サイズも一定のRNN系アーキテクチャです。1トークンあたりの計算量が一定で、状態サイズも固定——これは非常に魅力的な性質です。しかし、この「固定サイズ」という美点が同時に最大の弱点でもありました。状態が小さいと、長い文脈の中の情報を十分に覚えていられないのです。

Linear RNNの固定状態サイズは効率性の源であると同時に、長文脈リコールを阻む「記憶の壁」でもある。

2背景：DeltaNetと状態サイズのトレードオフ

本論文が拡張のベースとするのはGated DeltaNetデルタ則を使った更新とゲーティングを組み合わせたLinear Attention系モデルです。DeltaNetは「新しい情報を書き込む際、既存の内容を差し引いてから書く」というデルタ則的な更新を採用しており、Linear Attentionの中では比較的高性能なモデルです。

DeltaNetの内部状態は「Key × Value の外積2つのベクトルを掛け合わせて行列を作る演算。KVストアの核」で表現される密な行列です。状態サイズを大きくすれば長文脈の記憶は改善しますが、計算量（FLOPs）がそのまま増えてしまいます。「性能を上げたければ計算も増える」という、よくあるトレードオフです。

3提案手法：Sparse Delta Memory（SDM）の全体像

SDMのアイデアは一言でいうと「大きな外部メモリを用意して、必要な場所だけ読み書きする」です。密な行列全体を更新する代わりに、スパース（疎）なアドレッシングで巨大なメモリのごく一部だけを操作します。冷蔵庫の例で言えば、従来は小さな冷蔵庫に全部詰め込んでいたのが、SDMでは大きな倉庫を持ちつつ「今使うものだけ取り出す・しまう」というイメージです。

# 入力トークンごとに実行されるSDMの更新処理
1q, k, v, gate = project(x)①
# スパースアドレッシング：大きなメモリの一部を選択
2top_indices = sparse_select(k, memory)②
3mem_slice = memory[top_indices]
# デルタ則的な書き込み（既存内容を差し引いてから更新）
4delta_v = v - read(mem_slice, k)③
5mem_slice = mem_slice + outer(k, delta_v)
6memory[top_indices] = gate_update(mem_slice, gate)
# 読み出し：必要な情報だけ取り出して次の処理へ
7output = read(memory[top_indices], q)④
SDMの処理フロー（擬似コード）

① 入力の射影

通常のTransformerと同じくクエリ(q)・キー(k)・バリュー(v)に線形変換します。加えてSDMはゲート信号も生成します。

② スパースアドレッシング

キーベクトルkを使って、巨大なメモリの中から「関係のある」行だけをトップk個選びます。全体を触らないのでFLOPsが少なくて済みます。これが効率性の核心です。

③ デルタ則的な書き込み

選んだスライスに書き込む際、まず「今そこに何が書いてあるか」を読んでから差分（デルタ）だけ足します。これがDeltaNetゆずりのデルタ則更新です。「上書き」ではなく「修正」なので既存の情報を保ちながら更新できます。

④ 読み出し

クエリqを使って必要な情報をメモリから取り出し、次の層へ渡します。膨大なメモリを持ちながら、読み書きはスパースなので計算量は抑えられます。

4ボーナス機能：パラメトリック記憶としての初期状態学習

SDMには面白い追加機能があります。メモリの初期状態を学習可能なパラメータとして扱うという仕掛けです。通常、RNNの初期状態はゼロベクトルで固定されますが、SDMでは「学習によって良い初期値を見つける」ことができます。

これはいわば「ウォームアップした記憶」を最初から持ち込む感覚です。一般常識や推論に必要な知識をモデルの初期メモリに焼き付けておくことができ、パラメトリックメモリモデルのパラメータ自体が知識を記憶している仕組みとして機能します。論文ではこれにより「一般常識や推論タスクでもさらなる改善が見られた」と述べています。

6なぜ重要か：何が新しいのか

新規性 1

Dense（密）な状態更新をSparse（疎）なアドレッシングに置き換えることで、計算量を増やさずに状態容量を「桁違い」に拡大。FLOPs vs 容量のトレードオフを実質的に打破した。

新規性 2

Gated DeltaNetの更新則をスパースメモリに自然に拡張。デルタ則の「差分書き込み」をそのまま大きなメモリスライスへ適用するという、シンプルかつエレガントな設計。

新規性 3

メモリ初期状態の学習という「パラメトリック記憶」のアイデアを導入。長文脈リコールだけでなく、知識・推論タスクまで恩恵が波及する汎用性の高い設計。

新規性 4

Linear RNNとTransformerの中間的な存在を作ることで、「効率的だが記憶が弱い」というLinear RNNの根本的な欠点に対して理論的・実験的な解答を示した。

7意義と限界：正直に見ておこう

限界 1

スパースアドレッシングは「どこを読み書きするか」の選択が重要で、その選択の質が性能を左右する。選択メカニズムの設計次第で結果が大きく変わりうる。

限界 2

メモリが巨大になるとメモリ帯域（データの転送速度）がボトルネックになる可能性がある。FLOPsは抑えられてもハードウェア実装の難しさは残る。

限界 3

スパース操作はGPUで密行列演算ほど効率的に実装しにくい場合がある。実際のスループット（処理速度）はFLOPs計算だけでは測れない側面がある。
