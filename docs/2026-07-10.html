Sparse Delta Memory: Scaling the State of Linear RNNs through Sparsity
Loïc Cabannes, Pierre-Emmanuel Mazaré, Gergely Szilvasy, Matthijs Douze, Maria Lomeli, Ilze Amanda Auzina, Justin Carpentier, Gabriel Synnaeve, Hervé Jégou arXiv cs.LG ・ 2026年7月 原文 →

1なぜ「記憶」の設計がLLMの次の戦場なのか

みなさんがこれから研究するLLMの多くは、Transformer入力全体を相互に見比べながら処理するモデル構造。今のLLMの主流 という仕組みの上に成り立っています。その中核にある通常の(softmax)アテンションは、文章が長くなるほど「これまで出てきた単語すべて」を毎回参照し直すため、計算量とメモリ使用量が文脈の長さに比例して増えてしまいます。

これに対して近年注目されているのが、線形アテンションや状態空間モデルSSM。系列全体を固定サイズの内部状態に圧縮しながら処理するモデル(まとめて「linear RNN」系と呼ばれます)です。これらは過去の情報を固定サイズの「状態(state)」に圧縮して持ち運ぶため、文がどれだけ長くなっても1トークンあたりの計算量とメモリは一定になります。ところがこの固定サイズの状態こそが弱点でもあります。情報を詰め込む箱の大きさが決まっているので、詰め込みすぎると古い情報が薄れ、長い文脈から特定の事実を正確に思い出す(recall)性能でTransformerに劣ってしまうのです。

状態を大きくすれば思い出せる情報は増えるが、その分だけ計算コスト(FLOPs)も比例して膨らんでしまう――これが linear RNN 系がずっと抱えてきたジレンマだ。

2背景: 状態はどうやって更新されているか

Gated DeltaNet:「1枚のホワイトボード」に書き込み続ける仕組み

今回の論文が拡張している Gated DeltaNetlinear RNN系アーキテクチャの一つ。忘却ゲート付きのデルタルールで状態を更新する は、各トークンが持つ key と value のペアを、固定サイズの行列(状態)に外積という形でどんどん足し込んでいきます。デルタルール新しい情報で古い状態を少しずつ上書き更新する仕組み。最小二乗法に近い発想 によって、矛盾する情報は打ち消され、関連する情報は強め合うように更新されます。

たとえるなら、1枚の大きなホワイトボードに次々と要点を書き込んでいくようなものです。ボードの面積(状態サイズ)は決まっているので、書き込む量が増えるほど文字は重なり合い、古いメモは読み取りにくくなっていきます。状態を大きくする(=ボードを広げる)のがこれまでの素直な解決策でしたが、それは同時に「毎回ボード全体を書き換える」コストの増大を意味していました。

3手法: Sparse Delta Memory (SDM) というアイデア

著者らの発想はシンプルです。「ボード全体を毎回書き換える」のをやめて、代わりにもっと巨大な書庫を用意し、トークンごとに関係する棚だけを疎(スパース)に読み書きする、というものです。SDM は Gated DeltaNet の「密な外積による状態更新」を、疎なアドレッシング全体ではなく一部の要素だけを選んで読み書きする方式。計算量を抑えられる を使った巨大な明示的メモリへの読み書きに置き換えます。

この発想の利点は、メモリの総容量(=書庫の棚の数)をどれだけ増やしても、1トークンあたりに実際にアクセスする棚の数は変わらない、という点にあります。つまり記憶容量と計算コストを切り離せる。これは Gated DeltaNet のような従来手法が「状態を大きくする=毎回の計算量も増える」という制約から逃れられなかったのとは対照的です。

# 【従来】Gated DeltaNet: 状態行列(d×d)全体を毎トークン上書き
1state = dense_update(state, k, v)
# 【SDM】巨大な明示的メモリを用意し、学習可能な初期状態から始める
2memory = init_learned_memory()①
3for token in sequence:
4k, v, q = project(token)
# 全棚ではなく、関連する少数の棚だけを選ぶ
5slots = sparse_address(k, memory)②
6memory[slots] = delta_update(memory[slots], k, v)③
7output = read(memory[slots], q)
SDMの発想を単純化した擬似コード(論文の実装そのものではなく概念的なイメージ)

① 学習可能な初期メモリ ―― 「空の書庫」ではなく「仕込み済みの書庫」から始める

通常のRNNの状態は学習開始時にゼロで初期化されます。SDMでは初期メモリの中身自体を学習パラメータとして扱います。つまり事前学習の過程で「よく使う知識」をあらかじめ書庫に仕込んでおけるということです。これは長い文脈を扱う場面だけでなく、一般知識や推論タスクの性能向上にも効いてきます。

② 疎なアドレッシング ―― 毎回すべての棚を調べない

各トークンについて、メモリ全体ではなく関連しそうな一部の棚だけを選んで読み書きします。鍵になるのは、棚の総数(=記憶容量)をどれだけ増やしても、1トークンあたりに触る棚の数は増やさずに済む、という設計です。だから「巨大な書庫」を用意しても計算コスト(FLOPs)は増えません。

③ デルタ更新 ―― 選んだ棚だけを賢く書き換える

選ばれた棚に対しては、Gated DeltaNet と同じデルタルールの考え方で更新します。新しい情報と矛盾する古い情報を打ち消しつつ、関連する情報を強化するように、選ばれた棚の中身だけを更新する。全体を毎回書き換えるのではなく、狭い範囲を賢く更新するイメージです。

5なぜこの研究が重要なのか

LLMの性能向上はこれまで「モデルを大きくする」「計算量を増やす」という軸で語られることが多くありました。SDMが示すのは、記憶容量というもう一つの軸を、計算コストからほぼ切り離してスケールできる可能性です。これは長文脈処理や、エージェントが長時間タスクを遂行する際に必要な「経験の記憶」など、今後のLLM研究にとって重要な方向性の一つになりうるでしょう。

6意義と限界

視点1

アブストラクトからは、疎なアドレッシングを実際にどう実装しているか(ハッシュ方式や衝突の扱いなど)は詳しく分からない。この設計選択が性能や学習の安定性にどう効くのかは、論文本文で確認する価値がある。

視点2

疎な読み書きは、GPUのような並列ハードウェア上では効率的に実装しづらい処理になりがちです。isoFLOP比較だけでは、実運用での推論速度の向上がどれだけになるかまでは分かりません。

視点3

評価は Gated DeltaNet という特定の linear RNN アーキテクチャを土台にしています。他系統の linear RNN や Transformer とのハイブリッド構成にどこまで一般化するかは、今後の検証が必要な点でしょう。
