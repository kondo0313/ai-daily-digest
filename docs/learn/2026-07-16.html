Week 3: Transformer ─ 現代 LLM の心臓部 基礎〜中級

1なぜ Transformer は「革命」と呼ばれるのか

2017 年、Google の論文「Attention Is All You NeedRNN を使わず Attention だけで seq2seq を実現した革新的論文」が発表されました。それまでの機械翻訳は RNN や LSTM を積み重ねるアーキテクチャが主流でしたが、Transformer は 逐次処理を捨て、全トークン間の関係を一度に計算する設計を採用。学習の並列化と長距離依存の捕捉を同時に実現しました。

今日は Transformer の全体図を「工場の生産ライン」にたとえながら、Encoder・Decoder・FFN それぞれの役割を数式レベルまで掘り下げます。GPT や BERT など現代の LLM はすべてこのアーキテクチャを起源としているため、ここをしっかり理解することが今後の研究の土台になります。

Transformer は「逐次処理を並列処理に置き換えた」アーキテクチャ革命であり、現代 LLM すべての祖先である。

2Transformer の全体構造 ─ 工場のたとえ

Transformer を「精密部品の製造工場」として考えてみましょう。原材料（入力文）を受け取る 検査・前処理ライン（Encoder）、完成品（出力文）を一語ずつ組み立てる 組み立てライン（Decoder）、そして各ステーションで部品を非線形変換する 加工機（FFN） という三者の分業で成り立っています。

入力トークン列 $x_1, \dots, x_n$ はまず Embedding + Positional Encoding によってベクトル列に変換されます。これが工場への「原材料の搬入」に相当します。その後 Encoder と Decoder が連携して意味の理解と出力生成を行います。

3数学的基礎 ─ Scaled Dot-Product Attention から全体フローへ

Positional Encoding：位置情報の埋め込み

Transformer は RNN と違い順序を自動では知りません。そこで Positional Encodingトークンの位置情報を正弦波で数値化してベクトルに加算する手法 を用います。

Positional Encoding の定義（pos: トークン位置、i: 次元インデックス）
埋め込みベクトルに加算
これにより語の意味と位置情報が合成される

Scaled Dot-Product Attention

Transformer の中心にある計算です。Query・Key・Value という三つの行列を使い、各トークンが他のすべてのトークンをどれだけ「参照すべきか」を重みとして計算します。

Scaled Dot-Product Attention の定義（d_k: Key の次元数）
Q, K, V は入力 X から線形変換で生成
スコア行列（スケーリング前）
√d_k で割ることで内積の分散を安定化
softmax で各行を確率分布に変換
Attention の出力：V の加重和

Multi-Head Attention

単一の Attention は一種類の「視点」しか持てません。Multi-Head Attention複数の Attention を並列に走らせ結果を結合することで、多様な依存パターンを同時に捕捉する手法 を使うことで、文法・意味・位置など複数の観点を並行して学習できます。

Multi-Head Attention の定義（h 個のヘッドを結合して線形変換）
各ヘッド i の計算
h=8 のとき各ヘッドの次元
出力射影（全ヘッドを d_model に圧縮）

4Encoder の役割 ─「意味の地図」を作る

Encoder は N 個の同一構造の層を積み重ねたものです。各層は (1) Multi-Head Self-Attention と (2) Feed-Forward Network (FFN) の二つのサブ層からなり、それぞれに Residual Connection層をスキップして入力を出力に加算する接続。勾配消失を防ぐ と Layer Normalization が付きます。

Encoder 1 層の計算フロー（MHA: Multi-Head Self-Attention）
Self-Attention サブ層
FFN サブ層

Encoder は入力文の すべてのトークン間の関係を双方向に参照 できます。たとえば「銀行（bank）」という単語が「お金」の文脈なのか「川岸」の文脈なのかを、前後のトークンを見て判断する能力が Self-Attention によって実現されます。これが工場における「原材料の精密検査・前処理」です。

5Decoder の役割 ─「一語ずつ組み立てる」生成器

Decoder も N 層積み重ねですが、Encoder と異なり 三つ のサブ層を持ちます。

STEP 1

Masked Multi-Head Self-Attention：未来のトークンを見ないようマスクをかけた Self-Attention。生成時の「カンニング防止」に相当。

STEP 2

Cross-Attention（Encoder-Decoder Attention）：Query は Decoder 側、Key・Value は Encoder の出力を使う。「原材料の情報を参照しながら組み立てる」工程。

STEP 3

Feed-Forward Network (FFN)：各トークン位置で独立した非線形変換を適用。

Cross-Attention：Decoder が Encoder の出力を参照する計算
Q は Decoder の現在の表現
K, V は Encoder の最終出力
💡 Masked Attention の仕組み
マスクは上三角行列 M（対角より右上が -∞）を Attention スコアに加算することで実装します。softmax 後に -∞ の位置は 0 になるため、未来のトークンへの Attention が完全に遮断されます。 M[i][j] = 0 if j ≤ i else -∞ Attention = softmax((QKᵀ + M) / √d_k) · V

6FFN（Feed-Forward Network） ─「各ポジションの加工機」

Transformer の FFN はシンプルな 2 層の全結合ネットワークです。しかし 全トークンで重みを共有しながらも、各トークン位置で独立して適用 されるという特徴があります。Attention が「どのトークンに注目するか」を決めるのに対し、FFN は「注目した情報をどう変換するか」を担当します。

Transformer における Position-wise FFN の定義（ReLU 活性化）
第 1 層：d_model → d_ff に拡張（d_ff = 4 × d_model）
第 2 層：d_ff → d_model に圧縮
元論文の次元数（d_model=512, d_ff=2048）

Attention が トークン間の関係（水平方向の情報統合）を担うのに対し、FFN は 各トークン内部の表現変換（垂直方向の非線形変換）を担います。最近の研究では FFN が「知識の貯蔵庫」として機能しているという仮説も提唱されています。

⚙️ 現代 LLM の FFN 変種
GPT-2 以降は ReLU の代わりに GELU 活性化関数が主流です。また LLaMA などでは SwiGLU（Swish-Gated Linear Unit）という 3 行列版 FFN が採用され、より豊かな非線形変換を実現しています。 FFN_SwiGLU(x) = (Swish(xW₁) ⊙ (xW₃)) · W₂

7LLM への接続 ─ BERT と GPT は Transformer のどの部分を使うか

現代の LLM は Transformer を Encoder のみ または Decoder のみ で使うことが多いです。

Encoder only

BERT 系モデル。双方向 Self-Attention で文の理解・分類に強い。翻訳より「文書理解タスク」向き。

Decoder only

GPT 系・LLaMA 系。Masked Self-Attention で自己回帰的にテキストを生成。現在の ChatGPT もこの系統。

Encoder + Decoder

T5・BART 系。翻訳・要約など入力と出力の言語が異なるタスクに強い。元論文の完全な形。

💡 R&D でよく出る考え方：なぜ Decoder only が主流になったか
Decoder only モデルは「次のトークンを予測する」という単純な目標（言語モデリング）だけで学習できるため、ラベルなし大規模テキストをフル活用できます。これが GPT 系が急速にスケールできた理由の一つです。Encoder only は事前学習タスクの設計が複雑（MASK トークンの導入など）で、スケーリングで不利になりました。

8実装のヒント ─ Transformer Encoder Layer を実装する

1class EncoderLayer:
2def __init__(self, d_model=512, n_heads=8, d_ff=2048, dropout=0.1):
3self.self_attn = MultiHeadAttention(d_model, n_heads)
4self.ffn = FFN(d_model, d_ff) # 2層 ReLU / GELU
5self.norm1 = LayerNorm(d_model)
6self.norm2 = LayerNorm(d_model)
7self.dropout = Dropout(dropout)
8
9def forward(self, x, mask=None):
10# --- Sublayer 1: Multi-Head Self-Attention + Residual ---
11attn_out = self.self_attn(x, x, x, mask) # Q=K=V=x
12x = self.norm1(x + self.dropout(attn_out))
13# --- Sublayer 2: FFN + Residual ---
14ffn_out = self.ffn(x)
15x = self.norm2(x + self.dropout(ffn_out))
16return x # shape: (batch, seq_len, d_model)

⚙️ Pre-LN vs Post-LN
元論文は Post-LN（Attention → Add → Norm の順）ですが、現代の LLM の多くは Pre-LN（Norm → Attention → Add）を採用します。Pre-LN は学習が安定しやすく、大規模モデルでは標準になっています。疑似コードを書くときはどちらか明記する習慣をつけましょう。

9全体フローの整理 ─ 入力から出力まで一気通貫

STEP 1

入力トークン列 → Token Embedding + Positional Encoding → x̃ ∈ ℝ^(n × d_model)

STEP 2

x̃ を N 層の Encoder に通す → 文脈を含んだ表現 H_enc ∈ ℝ^(n × d_model)

STEP 3

出力トークン列（右シフト）→ Embedding + PE → ẙ ∈ ℝ^(m × d_model)

STEP 4

ẙ を N 層の Decoder（Masked SA → Cross-Attention with H_enc → FFN）に通す → H_dec

STEP 5

H_dec → Linear → Softmax → 次のトークンの確率分布 P ∈ ℝ^(m × |V|)

STEP 6

学習時は Cross-Entropy Loss で P と正解トークンを比較してパラメータ更新

Transformer の学習損失：各ステップの条件付き対数尤度の和
P はモデルのソフトマックス出力
全ステップを並列計算できるのが RNN との大きな違い

10まとめ ─ Transformer を「三つの役割」で覚える

本日の核心は Encoder = 理解、Decoder = 生成、FFN = 変換 という役割分担です。Attention が「どこを見るか」を決め、FFN が「見た情報をどう料理するか」を決める。この二つの組み合わせが Transformer の強さの源です。
