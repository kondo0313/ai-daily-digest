Week 2: ニューラルネットワーク基礎 ─ 層を重ねる意味 基礎〜中級

1なぜ「正規化」が必要なのか？ ─ 深いネットの悩み

料理に例えてみましょう。塩加減が毎回バラバラなシェフのレシピを覚えようとすると、「今日は塩辛い」「明日は薄い」と脳が混乱しますよね。ニューラルネットワークも同じです。層を重ねるほど、各層への「入力の分布」が学習中にどんどん変化してしまい、後続の層は「動くターゲットを追いかける」ような状況に陥ります。

この現象を内部共変量シフト学習中に各層への入力分布が変化し続ける問題。後段の層の学習を不安定にすると呼びます。2015年にGoogleのIoffe と Szegedy が提案したBatch Normalization（バッチ正規化）は、この問題を正面から解決し、深いネットワークの学習を劇的に安定・高速化しました。

2Batch Normalization の数学的な核心

Batch Norm のアイデアはシンプルです。「ミニバッチ内で各特徴量を平均0・分散1に揃えてから次の層に渡す」こと。ただし、後でネットワーク自身が「どの程度スケールやシフトを復元するか」を学習します。

Step 1: ミニバッチの平均を計算 (m はバッチサイズ)
バッチ内の分散
正規化（ゼロ平均・単位分散）
スケール・シフト（学習可能パラメータ γ, β）

ここで ε（イプシロン） は分母がゼロになるのを防ぐ小さな定数（例: 1e-5）です。γ（ガンマ）とβ（ベータ） はバックプロパゲーションで学習されるアフィン変換パラメータスケールとシフトを担う学習可能な係数。正規化の「やりすぎ」をネット自身が補正できるです。

「正規化で情報を壊す → γ・β で必要な表現力を取り戻す」この二段構えが Batch Norm の本質。

推論時はどうなる？ ─ 移動平均の活用

学習時はミニバッチの統計量を使いますが、推論時には「バッチ」という概念がありません。そこで学習中に指数移動平均過去の統計量を指数的に減衰させながら蓄積した平均・分散の推定値を記録しておき、推論時はその固定した値を使います。

指数移動平均で推論用の統計量を蓄積 (α は momentum、典型値 0.1)
推論時の正規化 (固定統計量を使用)

3直感で掴む ─ 「全員で足並みを揃える」 vs 「自分の中で揃える」

Batch Norm を学校のクラス（バッチ）に例えると、「クラス全員のテスト点数を平均・分散で正規化する」イメージです。クラス全体を見渡すので、バッチサイズが大きいほど安定します。逆にバッチが小さいと統計量の推定がブレ、学習が不安定になります。

一方、2016年に Ba らが提案した Layer Normalization（レイヤー正規化） は「一人の生徒の全科目の点数を正規化する」イメージです。Layer Normバッチ方向ではなく特徴量（チャネル）方向で正規化する手法。バッチサイズに依存しないは、バッチサイズが1でも機能するため、RNN・Transformer では標準的な選択肢になっています。

4Layer Normalization の数式

Layer Norm: サンプル i の H 個の特徴量にわたって平均を計算
特徴量方向の分散
正規化
スケール・シフト (γ, β は特徴量ごとに学習)

Batch Norm と構造的には酷似していますが、どの軸で平均・分散を取るかが決定的な違いです。Batch Norm はバッチ方向（サンプル間）、Layer Norm は特徴量方向（一サンプル内）で統計を計算します。

5各手法の正規化軸を図解で比較

Batch Norm

軸: バッチ × 空間。各チャネル（特徴マップ）ごとに、バッチ内の全サンプルをまとめて正規化。CNN に強い。バッチサイズが小さいと不安定。

Layer Norm

軸: 特徴量方向。1サンプルの全特徴量を正規化。バッチサイズ非依存。RNN・Transformer（GPT, BERT）で標準採用。

Instance Norm

軸: 空間方向のみ（サンプル・チャネルごと）。スタイル転送などの画像生成モデルで使用。

Group Norm

軸: チャネルをグループ分けして正規化。Batch Norm と Layer Norm の中間。小バッチでも安定した物体検出モデル（Mask R-CNN等）で活用。

6深掘り ─ なぜ学習が速くなるのか

勾配消失・爆発を抑える効果

勾配消失深い層ほど勾配が指数的に小さくなり、重みがほぼ更新されなくなる問題や勾配爆発は、活性化の分布が偏ることで悪化します。正規化によって各層の出力が適切なスケールに保たれると、誤差逆伝播での勾配も安定した大きさを保ちやすくなります。

Batch Norm の逆伝播における入力への勾配 (論文 Ioffe & Szegedy 2015 より)

この式が示す重要な点は、勾配が σ_B の逆数でスケールされることです。分散が大きい（活性化がバラバラ）なら勾配は小さく、分散が小さい（活性化が集中）なら勾配は大きくなり、自然な自己調整機能が生まれます。

学習率を大きくできる ─ 損失面の平滑化

MIT の Santurkar らの2018年の研究では、Batch Norm の真の効果は内部共変量シフトの解消よりも「損失景観（Loss Landscape）の平滑化」にあると示しました。正規化によって損失曲面が滑らかになり、大きな学習率でも安全にパラメータ空間を移動できるようになります。

💡 R&Dでよく出る考え方
Batch Norm の導入後は学習率を通常より5〜10倍大きく設定できる場合があります。ハイパーパラメータ探索のときは正規化の有無で学習率のスケールを分けて考えましょう。また、Batch Norm の直前に Bias を持つ層（Linear, Conv）がある場合、そのバイアスは正規化で打ち消されるため、bias=False にするのが実装上の定石です。

7LLM・現代 AI との接続 ─ Transformer は Layer Norm 一択？

GPT-2 以降の大規模言語モデルでは、Batch Norm ではなく Layer Norm が使われています。理由は明確です。言語モデルの学習では系列長が可変で、バッチサイズも小さいことが多く、バッチ方向の統計量が不安定になりやすいためです。さらに重要なのが Pre-LN と Post-LN の違いです。

Post-LN（元の Transformer）

残差接続の後に Layer Norm を適用。x → Attention → x + Attention(x) → LayerNorm。深いモデルでは勾配消失が起きやすい。

Pre-LN（GPT-2, GPT-3, LLaMA）

各サブ層の前に Layer Norm を適用。x → LayerNorm → Attention → x + Attention(LayerNorm(x))。学習が安定し、warm-up なしでも学習できる場合がある。現代の LLM の主流。

また、最近ではRMSNorm二乗平均平方根で正規化する Layer Norm の簡略版。平均を引く処理を省略し計算効率を高める（Root Mean Square Layer Norm）が LLaMA や Gemma シリーズで採用されています。

RMSNorm: 平均を引かず二乗平均平方根で割るシンプルな正規化

2
3# --- Batch Normalization ---
4# num_features = チャネル数（CNNの場合）
5bn = nn.BatchNorm1d(num_features=256)
6# バッチ方向で正規化: input shape (batch, 256)
7
8# --- Layer Normalization ---
9# normalized_shape = 正規化する軸の形状
10ln = nn.LayerNorm(normalized_shape=256)
11# 特徴量方向で正規化: input shape (batch, seq, 256)
12
13# --- Transformer ブロックの Pre-LN 実装例 ---
14class PreLNBlock(nn.Module):
15def __init__(self, d_model, nhead):
16super().__init__()
17self.norm = nn.LayerNorm(d_model)
18self.attn = nn.MultiheadAttention(d_model, nhead)
19
20def forward(self, x):
21# Pre-LN: 正規化 → Attention → 残差加算
22normed = self.norm(x)
23attn_out, _ = self.attn(normed, normed, normed)
24return x + attn_out # 残差接続

⚙️ ひとことメモ
PyTorch の BatchNorm には training フラグがあり、model.eval() にすると自動的に running_mean / running_var（移動平均の統計量）を使うよう切り替わります。model.train() / model.eval() の切り替えを忘れると、推論精度が大きくブレる原因になるので注意。これは面接でも頻出の「あるある落とし穴」です。

9各正規化手法の使い分けまとめ

CNN 画像分類

Batch Norm が第一選択。バッチサイズ ≥ 32 を確保できるなら安定して高性能。物体検出など小バッチなら Group Norm を検討。

RNN / LSTM

Layer Norm 一択。系列ステップごとに統計量が変わるため Batch Norm の適用が困難。

Transformer / LLM

Layer Norm（Pre-LN 配置）が標準。高速化・軽量化が重要なら RMSNorm も有力な選択肢。

拡散モデル / 画像生成

Group Norm または AdaLayerNorm（条件付き正規化）が多用される。

💡 R&Dでよく出る考え方
正規化手法の選択はアーキテクチャと密結合しています。「なぜこの論文はこの正規化を選んだか」を読み解くクセをつけましょう。バッチサイズ・系列長・分散学習環境の制約が選択理由に直結することがほとんどです。特に Data Parallelism でバッチをGPU間で分割する場合、Batch Norm の統計量をどう同期するかが性能のボトルネックになることがあります（Synchronized BN）。

10まとめ ─ 正規化は「土台を整える」技術

今日の核心を振り返りましょう。Batch Norm はバッチ内の統計量でミニバッチ全体の足並みを揃え、CNN の学習を劇的に安定・高速化しました。Layer Norm は一サンプル内の特徴量を正規化することでバッチサイズへの依存をなくし、Transformer という現代 AI の基盤技術を支えています。どちらも「正規化してから γ・β で表現力を取り戻す」という同じ思想に基づいています。

これらの正規化がなければ、GPT や BERT といった巨大モデルの学習は現実的に成立しなかったかもしれません。深く積み上げた層を「安定した土台」の上で動かす ─ それが正規化の果たす本質的な役割です。明日は残差接続（Residual Connection）と組み合わせた「深いネットワークがなぜ学習できるか」の謎に迫ります。

正規化は「制約」ではなく「自由」を与える技術 ─ 学習率を上げ、深さを増し、モデルの可能性を広げる。
