1「丸暗記」するネットワーク ─ 過学習という病

試験前日に問題集の答えをすべて丸暗記した学生を想像してください。翌日、まったく同じ問題なら満点が取れます。しかし少し形式が変わった途端、まったく歯が立たない。ニューラルネットワークでも同じことが起きます。これが 過学習訓練データには適合するが未知データには汎化しない現象（overfitting）です。

パラメータ数が数百万を超える深層モデルは表現力が非常に高く、少量の訓練データのノイズや偶然のパターンまで記憶しようとします。その結果、訓練損失は下がり続けるのに汎化誤差未知データに対する予測誤差は逆に増大するという逆転現象が生じます。今日は、この問題に立ち向かう三つの武器 ─ L2正則化・L1正則化・ドロップアウト ─ を数学から丁寧に解説します。

モデルが「記憶」ではなく「学習」するよう仕向けること ─ それが正則化の本質である。

2損失関数の復習と正則化項の登場

通常の学習では、予測と正解のズレを測る損失関数 $L(\theta)$ を最小化します。たとえば二乗誤差なら $L(\theta) = \frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2$ です。正則化は、この損失関数に パラメータ $\theta$ の大きさにペナルティを課す項 を加算するアイデアです。

正則化付き損失関数の一般形
$L(\theta)$: 元の損失
$\Omega(\theta)$: 正則化項（L1またはL2）
$\lambda > 0$: 正則化の強さを決めるハイパーパラメータ

直感的には「損失を小さくしたいが、パラメータが大きくなりすぎるのも嫌だ」という 二つの目標のトレードオフ を $\lambda$ で調整しています。$\lambda$ が大きいほど正則化が強く、モデルは単純な解を選びやすくなります。

3L2正則化（Weight Decay）の数学

勾配降下法への影響

L2正則化（Ridge正則化パラメータの二乗和をペナルティとして加える手法。重みの大きさを全体的に抑制する）では $\Omega(\theta) = \|\theta\|_2^2$ です。勾配を計算してみましょう。

L2正則化付き損失の勾配
更新則（勾配降下法）
整理すると「重みが毎ステップ少し縮む」ことがわかる

$(1 - 2\eta\lambda)$ の係数が $1$ より小さいため、勾配がゼロでも毎ステップ重みが少しずつ 縮小（decay） されます。これが「Weight Decay」と呼ばれる理由です。L2正則化はパラメータをゼロに近づけますが、完全にゼロにはしません。大きな重みを全体的に均等に抑制する効果があります。

ベイズ的解釈：ガウス事前分布

L2正則化はMAP推定（最大事後確率推定事前分布と尤度の積を最大化するパラメータ推定法）の観点からも理解できます。パラメータに対して $\theta_j \sim \mathcal{N}(0, \sigma^2)$ というガウス事前分布を仮定すると、対数事後確率を最大化することが、L2正則化付き損失の最小化と等価になります。

MAP推定とL2正則化の対応
ガウス事前分布を代入
$\lambda = \frac{1}{2\sigma^2}$ と対応し、分散が小さいほど正則化が強い

4L1正則化（Lasso）の数学 ─ スパースになる理由

L1正則化（Lasso正則化パラメータの絶対値の和をペナルティとする手法。一部の重みをちょうどゼロにするスパース解を生む）では $\Omega(\theta) = \|\theta\|_1$ です。絶対値は $\theta_j = 0$ で微分不可能なため、劣微分（subgradient） を用います。

絶対値関数の劣微分（subgradient）
ソフトしきい値演算子（proximal演算子）による解析解
結果：$|\theta_j| \leq \lambda$ の重みはちょうど0になる（スパース性）

L1正則化の最大の特徴は、小さな重みを ちょうどゼロ に落とすことです。これを スパース性 と呼びます。多数の特徴量から重要なものだけを自動的に選択する、特徴選択の効果を持ちます。L2が「全員を平等に少し削る」とすれば、L1は「弱いものはバッサリ切り捨てる」イメージです。

L2
全重みを均等に縮小・スパースにならない
L1
小さい重みをゼロに・自動的に特徴選択
Elastic Net
L1+L2の組み合わせ・両方の利点を享受

5幾何学的直感 ─ 制約領域で見る正則化

正則化は「制約付き最適化」としても解釈できます。L2正則化は $\|\theta\|_2^2 \leq c$ という 球（ハイパースフィア） の中で損失を最小化すること、L1正則化は $\|\theta\|_1 \leq c$ という ダイヤモンド形（超八面体） の中で最小化することに対応します。

L1の制約領域はダイヤモンドの角（頂点）を持ちます。損失関数の等高線がこの角にぶつかる確率が高く、角は座標軸上にあるため、必然的に $\theta_j = 0$ の成分が生まれます。一方、L2の球には角がないため、最適解はゼロになりにくいのです。この幾何学的構造がスパース解多くの成分がゼロのパラメータベクトルを生む根本的な理由です。

💡 R&Dでよく出る考え方
LLMの微調整（ファインチューニング）では、ほぼ必ずWeight Decay（L2）が使われます。AdamWオプティマイザの「W」はWeight Decayの意味で、勾配への適用ではなくパラメータへの直接的な減衰を行います。AdamとWeight Decayを正しく分離した点がAdamWの革新でした。

6ドロップアウト ─ ランダムに神経を「切る」技術

アルゴリズムの定義

2014年にSrivastavaらが提案した ドロップアウト訓練時にニューロンをランダムに無効化することで過学習を防ぐ正則化手法 は、シンプルながら強力です。確率 $p$（ドロップ率）で各ニューロンをゼロに置き換えます。

Inverted Dropoutの計算式（現代の標準実装）
$r_j$: ベルヌーイ確率変数（1-pの確率で1、pの確率で0）
$1/(1-p)$ でスケーリング：期待値を訓練・推論で一致させる

$(1-p)$ による除算（スケーリング）が重要です。これにより、ドロップアウトを 訓練時のみ 適用し、推論時はそのまま全ニューロンを使う ことができます。これを「Inverted Dropout」と呼び、現代のほぼすべての実装で使われています。

なぜ汎化するのか ─ アンサンブルの視点

ドロップアウトの深い理由は 暗黙的なアンサンブル学習 にあります。$n$ 個のニューロンがある層では、$2^n$ 通りのサブネットワークが考えられます。ドロップアウトは毎回ランダムに異なるサブネットワークを訓練することで、指数的に多くのモデルを共有パラメータで同時に学習しているとみなせます。

推論時は全ニューロンを使うことで、この膨大な数のサブネットワークの 幾何平均 に近い予測を行います。アンサンブル学習複数のモデルの予測を組み合わせてロバストな結果を得る手法は汎化性能が高いことが知られており、ドロップアウトはその恩恵を単一モデルで得られます。

共適応の防止

ドロップアウトのもう一つの効果は、ニューロン間の 共適応（co-adaptation） を防ぐことです。もし特定のニューロンAの出力に常に依存するニューロンBがあれば、AとBはセットで過学習パターンを覚えてしまいます。ドロップアウトにより「隣が消えても自分で機能する」ような 冗長で独立性の高い特徴表現 が育まれます。

p=0.5
全結合層の典型的なドロップ率
p=0.1〜0.2
Transformerでよく使われる低いドロップ率
2^n通り
nニューロンで訓練される暗黙のサブネット数

8LLM・現代AI研究への接続

大規模言語モデルの時代でも、これらの技術は現役です。GPT・BERTなどのTransformerは Attention層とFFN層の両方にドロップアウト を使います（ただし超大規模モデルでは $p$ を小さく設定する傾向があります）。また、ほぼすべてのファインチューニングレシピに AdamW（Weight Decay） が含まれており、L2正則化はデフォルトの慣習です。

近年では LoRA大規模モデルの一部のみを低ランク行列で微調整する効率的なファインチューニング手法 などのパラメータ効率的手法においても、L2正則化が重要な役割を果たします。モデルが小さな部分空間しか更新しない分、正則化によるバイアスが汎化に強く効くためです。

9まとめ ─ 三つの武器の使い分け

L2正則化 はすべての重みを均等に縮小し、滑らかで安定した解を与えます。実装が最も簡単でWeight Decayとして広く使われます。L1正則化 は不要な重みをゼロにするスパース性を持ち、特徴選択が必要な場面や解釈可能性が求められるときに有効です。ドロップアウト はランダム性を使った暗黙のアンサンブルで、特に全結合層の過学習を強力に防ぎます。

これら三つは排他的ではなく、組み合わせて使うのが実践的です。正則化は「制約」ではなく、モデルが「丸暗記から真の理解へ」脱却するための インセンティブ設計 です。明日はいよいよバッチ正規化とLayerNormへ進み、「データの流れを安定させる」もう一つの重要技術を学びます。

💡 R&Dでよく出る考え方
「正則化が強すぎる」場合も問題です（underfitting）。λや p を大きくしすぎると、モデルが十分な表現力を発揮できません。過学習（high variance）と未学習（high bias）のバランスを見ながら調整することが、実験設計の核心です。これをバイアス・バリアンストレードオフと呼びます。
