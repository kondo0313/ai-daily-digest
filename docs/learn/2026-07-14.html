1「どこを見るか」を学習する仕組み

人間が文章を読むとき、すべての単語に同じ力を注ぐわけではありません。「彼女はケーキを食べた。彼女は満足そうだった」という文を読むとき、2 文目の「彼女」が誰かを理解するために、自然と 1 文目の「彼女は」に目が引き寄せられます。これが「注意機構Attention。重要な箇所に自動的に重みを置いて情報を集める仕組み」の直感です。

2017 年に Google Brain が発表した論文「Attention Is All You NeedTransformer アーキテクチャを提案した画期的な論文。著者 Vaswani ら」は、この「注意」だけでシーケンスを処理できると示しました。その核心が Scaled Dot-Product Attention です。今日はその数学を一行ずつ丁寧に分解しましょう。

Attention とは「どの情報をどれだけ参照するか」を数値として学習する仕組みである。

2図書館の比喩 ─ Query・Key・Value の正体

Scaled Dot-Product Attention には 3 つの入力行列があります。Q（Query）・K（Key）・V（Value） です。これを図書館で考えてみましょう。あなたが司書に「機械学習の本はありますか？」と尋ねる検索ワードが Q。各本の背表紙に書かれたキーワードが K。実際に本を開いたときの内容が V です。

STEP 1

Query（Q）: 「今、自分が何を知りたいか」を表すベクトル。各トークンが持つ問い。

STEP 2

Key（K）: 「自分はどんな情報を持っているか」を表すベクトル。各トークンが持つ看板。

STEP 3

Value（V）: 「実際に取り出す情報」を表すベクトル。Key に対応する本当の内容。

STEP 4

Q と K の類似度を計算し、その重みで V を加重平均する。これが Attention の全体像。

3Scaled Dot-Product Attention の数式展開

では核心の数式を見ていきましょう。入力は Q ∈ ℝ^{n×d_k}、K ∈ ℝ^{m×d_k}、V ∈ ℝ^{m×d_v} です。n はクエリの数、m はキー/バリューの数、d_kQuery と Key のベクトル次元数。スケーリングの分母に使われる はキーの次元、d_v はバリューの次元です。

Scaled Dot-Product Attention の定義式
① 類似度行列の計算
② スケーリング（次元の爆発を防ぐ）
③ Softmax で確率分布へ変換（各行が確率）
④ Value の加重平均で出力を得る

なぜ √d_k で割るのか？ ─ スケーリングの数学的根拠

d_k が大きくなると内積 QK^⊤ の値も大きくなります。なぜでしょう？ q と k がそれぞれ平均 0・分散 1 の独立な要素を持つとき、内積 q · k は d_k 個の項の和なので分散加法性独立な確率変数の和の分散は各変数の分散の和に等しい性質 から分散が d_k になります。

内積の分散が d_k になることの確認（各要素が独立・平均0・分散1の場合）
標準偏差は √d_k → √d_k で割れば分散が1に正規化される

スケーリングなしで d_k が大きいと、内積値が非常に大きくなり Softmax が 極端に尖った分布（ほぼ 0 か 1）になります。すると勾配がほぼゼロになり、学習が止まってしまいます。√d_k で割ることで勾配消失を防ぐのです。

💡 R&D でよく出る考え方
Softmax の「温度（temperature）」という概念があります。Softmax(S/T) の T を大きくすると分布が均一に、小さくすると鋭くなります。Scaled Dot-Product の √d_k はまさにこの温度調整と同じ役割を果たしています。論文を読む際は temperature scaling として一般化して捉えると理解が深まります。

Softmax 関数の復習 ─ なぜ確率分布を作るのか

Softmax 関数の定義。出力はすべて正かつ合計が 1（確率分布）
数値安定化のため最大値を引く（実装上の工夫）

Softmax を使う理由は「どのトークンにどれだけ注目するか」を確率として解釈できるからです。出力 A_{ij} は「i 番目のクエリが j 番目のキーに注意を向ける割合」を表します。この値は 0〜1 の範囲で、行の合計が 1 になるため、Value の重み付き平均として自然に解釈できます。

Masked Attention ─ 未来を見ないための工夫

GPT のような自己回帰モデルでは、トークン i を予測するときに i より後ろのトークンを参照してはいけません（答えを見ながら予測することになる）。これを防ぐためにマスクを使います。

Causal Mask を加えた Masked Attention
M は上三角部分が -∞、下三角部分が 0 の行列
-∞ を加えると Softmax の出力がほぼ 0 になり未来を無視できる
⚙️ ひとことメモ
実装では -∞ の代わりに float('-inf') や -1e9 などの大きな負数を使います。-∞ をそのまま扱うと NaN が発生する場合があるため、実用上は -1e4 〜 -1e9 程度の値を使うことが多いです。PyTorch の torch.nn.functional.scaled_dot_product_attention にはこの処理が内包されています。

4Self-Attention と Cross-Attention ─ Q・K・V の由来

Q・K・V がどこから来るかによって Attention の種類が変わります。Self-Attention（自己注意） では Q・K・V がすべて同じ入力 X から生成されます。各トークンが系列内の他のすべてのトークンとの関係を計算できるため、文脈の把握が可能になります。

Self-Attention における Q・K・V の生成（X: 入力、W: 学習可能な重み行列）
W^Q, W^K ∈ ℝ^{d_model × d_k}、W^V ∈ ℝ^{d_model × d_v} が学習される

一方、Cross-Attention（交差注意） は Q をデコーダの状態から、K・V をエンコーダの出力から生成します。翻訳タスクでいえば「翻訳先の文が翻訳元のどこに注目すべきか」を計算するのがこれです。Encoder-Decoder入力を圧縮表現にするエンコーダと、それを展開して出力するデコーダの組み合わせ構造 型モデルでは両者が組み合わさります。

O(n²d)
Self-Attention の計算量（n: 系列長、d: 次元）
3
QKV 行列を生成するための線形変換の数
√d_k
スケーリング係数（Softmax の温度を安定化する）

5現代 LLM での使われ方

GPT-4・LLaMA・Claude などの大規模言語モデルはすべてこの Scaled Dot-Product Attention を積み重ねて構築されています。ただし原論文の形式からいくつかの改良が加えられています。例えば Multi-Head Attention異なる重み行列で複数の Attention を並列計算し結果を結合する手法 は d_k を h 個のヘッドに分割して多様な注意パターンを学習します。

また近年の研究では Flash AttentionGPU のメモリ階層を活用して Attention を高速化するアルゴリズム。Dao ら 2022 が注目されています。O(n²) のメモリ使用量を O(n) に削減しながら同一の計算結果を保証します。長文脈処理が求められる現代 LLM 研究では必須の知識です。

6実装のヒント ─ PyTorch で Attention を書く

1def scaled_dot_product_attention(Q, K, V, mask=None):
2# Q: (batch, n, d_k), K: (batch, m, d_k), V: (batch, m, d_v)
3d_k = Q.shape[-1]
4
5# STEP 1: 内積で類似度を計算
6scores = Q @ K.transpose(-2, -1) # (batch, n, m)
7
8# STEP 2: √d_k でスケーリング
9scores = scores / sqrt(d_k)
10
11# STEP 3: マスク（任意）
12if mask is not None:
13scores = scores + mask # mask=-inf で未来をブロック
14
15# STEP 4: Softmax で確率分布へ
16attn_weights = softmax(scores, dim=-1) # 行方向
17
18# STEP 5: Value の加重平均
19output = attn_weights @ V # (batch, n, d_v)
20return output, attn_weights

💡 R&D でよく出る考え方
Attention の重み行列（attn_weights）は可視化が非常に有益です。どのトークンがどのトークンに注目しているかを確認することで、モデルが文法・意味をどう処理しているか直感的に理解できます。BERTVisualize などのツールや、seaborn のヒートマップで簡単に可視化できます。研究では Attention の解釈可能性（Interpretability）がひとつの重要な課題になっています。

7計算量の理解 ─ なぜ長文脈は難しいか

Scaled Dot-Product Attention の計算量を整理しておきましょう。系列長 n、モデル次元 d_model とすると、QK^⊤ の計算は O(n² × d_k) です。n が 2 倍になると計算量が 4 倍になります。GPT-4 が 128K トークン、Claude が 200K トークンの長文脈に対応するには、この二乗コストをどう乗り越えるかが鍵です。

Attention の計算量・空間量。系列長の二乗がボトルネック
FlashAttention は再計算で空間量を O(n) に削減
⚙️ ひとことメモ
この O(n²) 問題を解決しようとした研究は多く、Linformer・Performer・Longformer などが代表例です。ただし現時点では FlashAttention + KV Cache の組み合わせが実用上最も広く採用されています。理論的な面白さと実用性のギャップに注目するのも研究者の目線です。

8まとめ ─ Attention が「万能」な理由

Scaled Dot-Product Attention の本質は「Q と K の内積で類似度を計り、その重みで V を集約する」という非常にシンプルな操作です。しかしこのシンプルさの中に、位置に依存しない長距離依存の捕捉・並列計算の容易さ・学習可能な重み行列による柔軟性という 3 つの強みが詰まっています。

明日の DAY 12 では、この Attention を h 個並列に走らせる Multi-Head Attention と、それを積み重ねる Transformer ブロック全体の構造に進みます。今日の数式 Attention(Q,K,V) = softmax(QK^⊤/√d_k)V をしっかり手に馴染ませておくと、残りの Transformer の議論がぐっとクリアになるはずです。
