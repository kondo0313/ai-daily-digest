1きょうの問い ─ 行列って結局なんですか？

「行列は数を並べた表」——そう習いましたか？ それは正しいですが、研究者にとって重要な直感は別にあります。行列は「空間を変換する機械」です。この視点の転換だけで、機械学習の多くの操作が「なぜそうなのか」と腑に落ちます。

今日は「行列 = 変換」という見方を徹底的に身につけ、固有値分解行列を固有値と固有ベクトルに分解する操作と特異値分解長方形行列を対角行列の形に分解する汎用的な手法（SVD）（SVD）まで理解します。これらは LLM の重み行列、LoRA、Attention のすべてに顔を出す基本道具です。

2動機 ─ なぜ AI 研究者に線形代数が必要か

Transformer の Self-Attention は、ざっくりいうと「行列の掛け算を何度も繰り返す」操作です。入力トークンの埋め込みベクトルに Query/Key/Value の重み行列をかけ、Attention スコアを計算し、Value で重み付き和を取る——すべて線形代数です。

また、LoRA大きな重み行列の更新を低ランクの小さな行列の積で近似するパラメータ効率チューニング手法の核心は「大きな行列を低ランクの積 $A \cdot B$ で近似できる」という線形代数の事実にあります。線形代数の直感なしに、これらの手法の「なぜ動くのか」を理解するのは困難です。

3行列 = 線形変換

まず「変換」とは何かを確認します。関数 $f: \mathbb{R}^n \to \mathbb{R}^m$ が線形であるとは、以下の2条件を満たすことです。

条件①

加法性: $f(\mathbf{u} + \mathbf{v}) = f(\mathbf{u}) + f(\mathbf{v})$ — 先に足してから変換しても、変換してから足しても同じ

条件②

斉次性: $f(c\mathbf{u}) = cf(\mathbf{u})$ — スカラー倍してから変換しても、変換してからスカラー倍しても同じ

この2条件を同時に書くと「重ね合わせの原理」になります。

線形性の条件（重ね合わせの原理）

そして「$n$ 次元空間から $m$ 次元空間への線形変換はすべて $m \times n$ 行列で表現できる」というのが、線形代数の根本定理です。逆に言えば、すべての行列は線形変換です。行列 $\mathbf{W}$ とベクトル $\mathbf{x}$ の積 $\mathbf{W}\mathbf{x}$ は、「$\mathbf{x}$ が表す点を $\mathbf{W}$ という変換機でねじ曲げて新しい点 $\mathbf{y}$ に送る」操作です。

行列-ベクトル積 = 線形変換
💡 ニューラルネットの全結合層とは
全結合層（Linear層）は $\mathbf{y} = \mathbf{W}\mathbf{x} + \mathbf{b}$ です。バイアス $\mathbf{b}$ がある分、厳密にはアフィン変換（原点を移動する線形変換）ですが、本質は線形変換です。$\mathbf{W}$ のランクや固有値が、この層が「どんな変換をしているか」を決めます。

行列の掛け算 = 変換の合成

行列 $\mathbf{A}$ と $\mathbf{B}$ の積 $\mathbf{AB}$ は「$\mathbf{B}$ で変換してから $\mathbf{A}$ で変換する」という変換の合成を一つの行列で表したものです。（右から左の順で読む点に注意。）

Transformer の Attention では、$\mathbf{Q} = \mathbf{X}\mathbf{W}_Q$, $\mathbf{K} = \mathbf{X}\mathbf{W}_K$, $\mathbf{V} = \mathbf{X}\mathbf{W}_V$ という3つの線形変換が入力トークン行列 $\mathbf{X}$ に適用されます。これはまさに「3回の変換を別々の重みで行う」操作です。

4固有値と固有ベクトル ─ 変換に「不動の軸」はあるか

行列 $\mathbf{A}$ という変換機は、ほとんどのベクトルをぐいっと別方向に曲げます。しかし特別なベクトル $\mathbf{v}$ があって、これだけは変換後も元の方向を向いたまま（スカラー倍されるだけ）のことがあります。

固有値方程式 — $\mathbf{v}$ は固有ベクトル、$\lambda$ は固有値
整理すると
非自明解が存在する条件

固有値 $\lambda$ が正なら「変換後も同じ向き（スケール変化のみ）」、負なら「向きが反転」、絶対値 $|\lambda| > 1$ なら「引き伸ばされる」、$|\lambda| < 1$ なら「縮む」軸です。

なぜ固有値が重要か

正方行列 $\mathbf{A}$ が n 個の線形独立な固有ベクトルを持つとき、行列は対角化できます。

固有値分解（対角化） — $\mathbf{P}$ の列が固有ベクトル、$\mathbf{\Lambda}$ が固有値の対角行列

これは「行列 $\mathbf{A}$ による変換は、①固有ベクトルを基底に変換し、②各軸を $\lambda_i$ 倍スケールし、③元の基底に戻す」という3ステップに分解できると言っています。直感的に非常にクリアです。

機械学習での応用は豊富です：主成分分析データの分散が最大となる方向（主成分）を固有ベクトルで求める次元削減手法（PCA）（PCA）は共分散行列の固有ベクトルを求める操作ですし、グラフニューラルネットワークのグラフラプラシアングラフの接続構造を行列表現したもの。固有ベクトルがスペクトルグラフ理論の基礎の固有ベクトルはグラフ構造の基底となります。

5特異値分解 (SVD) ─ 長方形行列の「真の姿」

固有値分解は正方行列にしか使えません。しかし、ニューラルネットの重み行列 $\mathbf{W} \in \mathbb{R}^{m \times n}$（$m \neq n$ が普通）には使えない。そこで登場するのが特異値分解（SVD）です。

直感から入りましょう。どんな線形変換でも、「①入力空間を回転し、②各軸をスケールし、③出力空間を回転する」という3ステップで書けます。これを式にしたのが SVD です。

特異値分解 — $\mathbf{U} \in \mathbb{R}^{m \times m}$、$\boldsymbol{\Sigma} \in \mathbb{R}^{m \times n}$（対角）、$\mathbf{V} \in \mathbb{R}^{n \times n}$（直交行列）
$\boldsymbol{\Sigma}$ の対角成分 $\sigma_1 \geq \sigma_2 \geq \cdots \geq 0$

$\mathbf{U}$（左特異ベクトル行列）と $\mathbf{V}$（右特異ベクトル行列）はどちらも直交行列転置が逆行列に等しい行列。幾何的には回転・反転を表すです（$\mathbf{U}^\top\mathbf{U} = \mathbf{I}$）。$\boldsymbol{\Sigma}$ は特異値 $\sigma_i$ が降順に並んだ対角行列です。

低ランク近似 ─ LoRA の数学的根拠

SVD の最も強力な応用が低ランク近似です。特異値 $\sigma_1 \geq \sigma_2 \geq \cdots$ のうち、大きいもの上位 $r$ 個だけを残して他を0にすると、元の行列 $\mathbf{W}$ を最もよく近似する「ランク $r$ の行列」が得られます。

低ランク近似 — $\mathbf{A} \in \mathbb{R}^{m \times r}$、$\mathbf{B} \in \mathbb{R}^{r \times n}$（$r \ll \min(m,n)$）

これがまさに LoRA（Low-Rank Adaptation）の設計思想です。大規模な重み行列 $\mathbf{W}$ の更新量 $\Delta\mathbf{W}$ を「低ランク $r$ の2つの小行列 $\mathbf{A}, \mathbf{B}$ の積」で近似します。なぜこれが動くのか？——「Fine-tuning によって重みに加わる変化は、実質的に低次元の構造を持つ」という仮説があり、SVD の低ランク近似がそれを捉えられるからです。

LoRA は「重みの更新は低ランク」という SVD の洞察を Fine-tuning に応用した発明だった。

Attention の行列乗算を具体的に追う

Self-Attention の計算を線形代数の言葉で書くと以下のようになります。$n$ トークン、$d$ 次元の入力行列 $\mathbf{X} \in \mathbb{R}^{n \times d}$ に対し：

1# ① 入力を Q/K/V 空間に線形変換（3回の行列積）
2Q = X @ W_Q
3K = X @ W_K
4V = X @ W_V
5# ② スコア行列（Q と K の内積）
6scores = Q @ K.T / sqrt(d_k)①
7# ③ Softmax で確率分布に
8attn = softmax(scores)
9# ④ Value の重み付き和（最終出力）
10output = attn @ V②
Self-Attention の線形代数ステップ（疑似コード）
⚙️ ①のスケーリングの意味
$\sqrt{d_k}$ で割るのは、内積が次元数 $d_k$ に比例して大きくなりSoftmax確率ベクトルに変換する活性化関数。大きな値に急激に集中するため温度管理が重要が極端な値（ほぼ one-hot）に偏るのを防ぐためです。これも線形代数と確率論の組み合わせです。

7きょうの到達点

💡 今日マスターしたこと
今日学んだことを確認しましょう。①行列 = 線形変換という視点（数の表ではなく空間の変換機）。②固有値・固有ベクトル（変換で方向が変わらない「不動の軸」）。③SVD = どんな行列でも分解できる究極のツール。④低ランク近似 = LoRA の数学的根拠。これらが LLM の Attention・Fine-tuning・圧縮の至るところで使われています。明日は「微積分とバックプロパゲーション」に進み、学習の仕組みを数式で理解します。
