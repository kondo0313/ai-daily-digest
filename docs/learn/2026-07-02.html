1なぜ「不確かさ」を数値化するのか？

天気予報を考えてみましょう。「明日は晴れか雨か五分五分」と言われるのと、「明日は 99% の確率で晴れ」と言われるのでは、どちらの予報の方が「情報量」が多いでしょうか？ 直感的には、当たり前のことを言っている予報よりも、意外性のある予報ほど多くの情報を伝えている気がします。この直感を数学的に厳密に定式化したのが情報理論であり、その核心概念が エントロピー確率分布の「不確かさ」や「乱雑さ」を一つの数値で表す量 です。

現代の AI・機械学習の研究では、エントロピーとその仲間たちが至るところに登場します。言語モデルの学習損失であるクロスエントロピー二つの確率分布間のズレを測る量で、予測分布が正解分布にどれだけ近いかを表す、生成モデルの評価指標 KL ダイバージェンス、強化学習の探索戦略など、これらをしっかり理解することが AI 研究者としての確固たる基盤になります。

「情報量」とは驚きの大きさ ─ 起こりにくいことほど、起きたときに多くの情報を運ぶ。

2確率と情報量 ─ 数学的基礎

自己情報量：1 つの事象が持つ情報量

事象 $x$ が確率 $p(x)$ で起こるとき、その自己情報量（surprisal）は次のように定義されます。確率が小さいほど情報量が大きくなるように、対数の逆数を使います。

自己情報量の定義。底を 2 にとると単位は bit、自然対数なら nat。
p(x)=1（確実）のとき
p(x)=1/2（コイン表）のとき
p(x)=1/1024 のとき
⚙️ ひとことメモ
底を 2 にするか自然対数にするかは慣習の問題。情報工学では bit（底 2）、機械学習の実装では nat（底 e）が多い。PyTorch の CrossEntropyLoss は自然対数を使用します。

シャノンエントロピー：分布全体の平均的な不確かさ

自己情報量は 1 つの事象に対するものでした。確率分布 $P$ 全体の「平均的な不確かさ」を求めるには、自己情報量の期待値確率で重み付けした平均値 を計算します。これが シャノンエントロピー $H(P)$ です。

シャノンエントロピーの定義。分布が均一なほど大きく、確定的なほど 0 に近づく。
コイン投げ（公平）p=(0.5, 0.5) のとき
コイン投げ（偏り）p=(0.9, 0.1) のとき
確定的な分布 p=(1, 0) のとき
0 bit
完全に予測可能な分布のエントロピー（下限）
log|X| bit
一様分布のエントロピー（上限）
~10.8 bit
日本語テキスト 1 文字あたりの情報量の目安

3直感でつかむ ─ エントロピーは「驚きの平均」

エントロピーをもっと直感的に理解するために、図書館のたとえを使いましょう。すべての本が全く同じ内容だったら（確定的な分布）、図書館から一冊取り出しても「知っている内容」なので驚きゼロ ─ エントロピー 0 です。一方、ランダムな内容の本が均等に並んでいたら（一様分布）、どの本を手に取っても最大の驚きがある ─ エントロピーは最大です。

言語モデルにとってのエントロピーも同じ意味を持ちます。次の単語を完璧に予測できればパープレキシティ言語モデルの評価指標。エントロピーを指数化したもので、平均的に何択から選んでいるかを表す は 1（驚きなし）、全く予測できなければパープレキシティは語彙数と同じ（最大の驚き）になります。

4クロスエントロピー ─ 予測のズレを測る

実際の分布（正解）を $P$、モデルが予測した分布を $Q$ とするとき、クロスエントロピー $H(P, Q)$ は「$P$ に従う出来事を、$Q$ を使って説明したときの平均情報量」です。

クロスエントロピーの定義。Q が P に近いほど H(P,Q) は H(P) に近づく。
Q = P のとき（完璧な予測）
Q が P から外れるほど

機械学習では、正解ラベルが one-hot ベクトル（$p(y^*) = 1$、他は 0）の場合、クロスエントロピー損失は非常にシンプルな形になります。

one-hot 正解ラベルに対するクロスエントロピー損失。正解クラスの確率の対数のみが残る。
q(y*)=0.9（自信を持った正解）
q(y*)=0.1（自信を持った不正解）

5KL ダイバージェンス ─ 分布間の距離

定義と直感

KL ダイバージェンス（Kullback–Leibler divergence）$D_{\text{KL}}(P \| Q)$ は、分布 $Q$ を使って分布 $P$ を近似したときの「情報の無駄」を測る量です。クロスエントロピーからシャノンエントロピーを引いたものと考えると直感が掴みやすいです。

KL ダイバージェンスの定義。「クロスエントロピー ー エントロピー」で、余分な情報量を表す。
期待値表現に書き換える
Q = P のとき（KL の最小値）
非負性（ギブスの不等式）

KL は「非対称」─ 距離ではなく乖離度

重要な注意点として、KL ダイバージェンスは一般に $D_{\text{KL}}(P \| Q) \neq D_{\text{KL}}(Q \| P)$ です。つまり数学的な意味での「距離」ではなく「乖離度」です。この非対称性は AI 研究では重要な意味を持ちます。

KL ダイバージェンスの非対称性。どちらの方向で測るかで意味が変わる。
Forward KL: P を Q で近似（P に従うデータを Q で説明）
Reverse KL: Q を P で近似（Q に従うデータを P で説明）
💡 R&Dでよく出る考え方
VAE（変分オートエンコーダ）の損失関数には Reverse KL が登場します。また RLHF（人間フィードバックによる強化学習）では、学習後のモデルが元のモデルから逸脱しすぎないように KL ペナルティ $D_{\text{KL}}(\pi_{\theta} \| \pi_{\text{ref}})$ を損失に加えます。どちらの方向の KL を使うかで学習の挙動が変わります。

7現代 AI における活用シーン

STEP 1

【言語モデルの訓練】GPT などの大規模言語モデルは、次トークン予測タスクでクロスエントロピー損失を最小化して学習します。「正解トークンの対数確率の平均」を最大化する操作です。

STEP 2

【生成モデル（VAE）】変分オートエンコーダの ELBO（証拠下限）は「再構成損失（クロスエントロピー）＋ KL ダイバージェンス」の形をしています。KL 項が潜在表現を正規分布に近づけます。

STEP 3

【RLHF・PPO】ChatGPT に使われた技術。強化学習でモデルを微調整する際、KL ダイバージェンスを正則化項として使い、学習が暴走（Mode Collapse）するのを防ぎます。

STEP 4

【知識蒸留】大きなモデル（教師）の出力分布 P と、小さなモデル（生徒）の出力分布 Q の KL ダイバージェンスを最小化することで、知識を転移します。

STEP 5

【情報ボトルネック・探索戦略】エントロピーを「探索の指標」として使う手法（Maximum Entropy RL など）は、強化学習の探索問題に広く応用されています。

8実装で確かめる ─ Python での計算

1import numpy as np
2from scipy.special import rel_entr
3
4# 確率分布を定義
5P = np.array([0.4, 0.4, 0.2]) # 真の分布（正解）
6Q = np.array([0.3, 0.5, 0.2]) # モデルの予測分布
7
8# シャノンエントロピー H(P)
9H_P = -np.sum(P * np.log2(P))
10print(f'H(P) = {H_P:.4f} bit')
11
12# クロスエントロピー H(P, Q)
13H_PQ = -np.sum(P * np.log2(Q))
14print(f'H(P,Q) = {H_PQ:.4f} bit')
15
16# KL ダイバージェンス D_KL(P || Q)
17KL_PQ = np.sum(rel_entr(P, Q)) / np.log(2) # nat→bit変換
18print(f'KL(P||Q) = {KL_PQ:.4f} bit')
19
20# 確認: H(P,Q) = H(P) + KL(P||Q)
21print(f'H(P) + KL = {H_P + KL_PQ:.4f} bit ← H(P,Q) と一致するはず')
NumPy と SciPy でエントロピー・KL ダイバージェンスを計算する
💡 R&Dでよく出る考え方
「クロスエントロピーを最小化する」=「KL ダイバージェンスを最小化する」であることを、この計算で実際に確かめてみてください。データ分布 P が変わらない限り、H(P) は定数なので、H(P,Q) を下げることと KL(P‖Q) を下げることは完全に等価です。これが分類問題の損失関数に CrossEntropy が使われる数学的な理由です。

9まとめと次への架け橋

今日は情報理論の三本柱を学びました。自己情報量（驚きの大きさ）→エントロピー（分布全体の平均的な驚き）→クロスエントロピー（予測の精度）→KL ダイバージェンス（分布間の乖離）という流れで、それぞれが自然につながっています。
