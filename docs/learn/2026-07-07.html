1なぜ「深さ」という概念が生まれたのか

画像を見て「猫だ」と判断するとき、あなたの脳は何をしているのでしょう。「ピクセルの輝度をすべて調べてからラベルを出力する」わけではないですよね。まずエッジを検出し、次にそのエッジが組み合わさったパーツ（目・耳・ひげ）を認識し、最後にそれらが統合されて「猫」という概念が浮かぶ ─ 情報は段階的に抽象化されていきます。多層パーセプトロン（MLP）複数の線形変換＋非線形活性化を積み重ねたニューラルネットワークは、まさにこの「段階的な抽象化」を計算で再現しようとする試みです。

深さとは「変換の回数」ではなく、「表現の抽象度が上がる段数」である。

2単層パーセプトロンの限界 ─ XOR 問題から考える

1969年、Minsky と Papert は著書 *Perceptrons* で衝撃的な事実を示しました。単層パーセプトロン（線形分類器）は XOR を学習できない、と。XOR の真理値表を2次元平面にプロットすると、出力が 1 になる点と 0 になる点を 1 本の直線で分けることが不可能だとわかります。

XOR は線形分離不可能な問題の最も単純な例
単層モデルの予測式
線形境界の限界

この限界を突破するには「中間表現」が必要です。1つ隠れ層入力層と出力層の間にある層。生のデータを変換した「特徴」を保持するを追加すると、入力空間を非線形に曲げることができ、XOR も正しく分類できるようになります。

3多層パーセプトロンの数学的定義

順伝播（Forward Propagation）の定式化

$L$ 層の MLP は以下のように再帰的に定義されます。各層 $l$ はアフィン変換と非線形活性化の合成です。

入力ベクトル（第0層の表現）
第 l 層のアフィン変換
非線形活性化の適用
最終出力（回帰の場合）

パラメータ総数は $\sum_{l=1}^{L}(d_l \cdot d_{l-1} + d_l)$ です。ここで重要なのは、$\phi$ が非線形でなければ何層積んでも線形変換に還元されるという点です。

活性化なしで層を重ねても結局は1つの線形変換
つまり隠れ層の意味がなくなる

4万能近似定理 ─ 理論的に何が保証されるか

万能近似定理（Universal Approximation Theorem）適切なサイズの1隠れ層ネットワークが任意の連続関数を近似できるという理論は、1989年に Cybenko（シグモイド活性化）、1991年に Hornik らによって一般化されました。

Cybenko の万能近似定理（1989）─ 単一隠れ層でも原理上は任意の連続関数を近似できる
ただし N は指数的に大きくなりうる
「1層で近似できる」は正しい。しかし「1層で効率的に近似できる」は間違い ─ それが深さを使う理由だ。

深さの効率性 ─ 表現の指数的優位

深いネットワークは、同じパラメータ数で浅いネットワークよりも指数的に多くの線形領域を作れます。ReLU ネットワークを例に取ると、深さ $L$・各層幅 $n$ のネットワークが作れる線形領域数は以下のように推定されます。

深いネットワーク（L 層）は指数的に増大

直感的なたとえ話で言えば、折り紙に似ています。紙を1回折るより何度も折ったほうが複雑な形が作れますよね。ReLU の各層は「特徴空間を折る」操作に相当し、層を重ねるほど「折り目」が指数的に増えるのです。

6深さが増すことのトレードオフ

勾配消失問題

バックプロパゲーションでは勾配は出力層から入力層へ連鎖律で伝わります。各層でシグモイドのような飽和する活性化を使うと、勾配が指数的に0へ近づく勾配消失問題誤差信号が深い層ほど消えてしまい重みが更新されなくなる現象が発生します。

連鎖律による勾配の計算 ─ 積の形になるため消失・爆発が起きやすい
シグモイドの勾配の最大値は 0.25
ReLU ならアクティブ領域で勾配は 1

解決策の系譜

STEP 1

ReLU 活性化関数の採用（2010年代〜）：勾配消失を大幅に緩和

STEP 2

バッチ正規化（Batch Normalization, 2015）：各層の活性化分布を安定させる

STEP 3

残差接続（ResNet, 2015）：層をスキップするショートカットで勾配を直接伝える

STEP 4

重み初期化の工夫（He 初期化など）：学習開始時の勾配スケールを適切に設定

7実装：2層 MLP で XOR を解く

1import numpy as np
2
3def relu(z):
4return np.maximum(0, z)
5
6def sigmoid(z):
7return 1 / (1 + np.exp(-z))
8
9# XOR データ
10X = np.array([[0,0],[0,1],[1,0],[1,1]]) # (4, 2)
11y = np.array([[0],[1],[1],[0]]) # (4, 1)
12
13# 隠れ層（2ユニット）の手動設定（学習後の理想的な重み）
14W1 = np.array([[1, 1], [1, 1]]) # (2, 2)
15b1 = np.array([[-0.5, -1.5]]) # (1, 2)
16W2 = np.array([[1], [-2]]) # (2, 1)
17b2 = np.array([[-0.5]]) # (1, 1)
18
19# 順伝播
20z1 = X @ W1 + b1 # 隠れ層: アフィン変換
21h1 = relu(z1) # 隠れ層: ReLU 活性化
22z2 = h1 @ W2 + b2 # 出力層: アフィン変換
23y_hat = sigmoid(z2) # 出力層: Sigmoid → [0,1]
24print(np.round(y_hat, 2)) # [[0.38],[0.62],[0.62],[0.38]] ≈ XOR

💡 R&Dでよく出る考え方
「何層にすべきか」に正解はありませんが、実務では まず小さなネットワークを試して過小適合を確認 → 徐々に深く・広くする のが王道です。深さを増やすよりも幅を増やすほうが最適化しやすいこともあります（Width vs Depth のトレードオフ）。
⚙️ ひとことメモ
万能近似定理は「存在性」を保証するだけで「学習できる」とは言っていません。勾配降下法で実際に解にたどり着けるかは最適化の問題であり、深さ・幅・初期化・学習率などが絡む別の議論です。

8現代 AI 研究への接続

GPT や ViT に代表される現代の大規模モデルは、数十〜百層超の深さを持ちます。表現の階層性の思想はそのまま生きており、Transformer の各層は「どの情報に注目するか（アテンション）」を段階的に洗練させています。残差接続（Residual Connection）入力をそのまま出力に足し合わせるショートカット。勾配消失を防ぎ深いネットワークを安定して学習させるはほぼすべての深層モデルに採用され、深さの恩恵を引き出す基盤技術になっています。

また、Neural Scaling Laws の研究（Kaplan et al., 2020）によって、パラメータ数・データ量・計算量を増やすほどモデル性能が冪乗則に従って改善することが実証されました。「深さ（と幅）が増すほど表現力が増す」という理論が、実際のスケールアップ戦略の根拠になっています。

9まとめ ─ 深さの本質

①

単層パーセプトロンは線形分離不可能な問題を解けない。XOR がその典型例。

②

万能近似定理は「1層でも原理上は何でも近似できる」と言うが、必要なユニット数が指数的に爆発する。

③

深さを増すと線形領域数が指数的に増加し、同じパラメータ数でより複雑な関数を効率的に表現できる。

④

各層は低レベル→高レベルへと段階的に抽象化された表現を学習する（階層的特徴学習）。

⑤

勾配消失という深さの代償は、ReLU・BatchNorm・残差接続によって克服されてきた。

「なぜ深くするのか」─ それは、自然界の複雑さを階層的に分解し、有限のパラメータで指数的に豊かな表現を得るためである。
