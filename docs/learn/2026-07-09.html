1「なぜ二乗誤差ではダメなのか」から始める問い

ニューラルネットワークを学び始めると、分類問題では「クロスエントロピー損失モデルの出力確率と正解ラベルのズレを情報理論的に測る損失関数」を使うよう言われます。しかし、なぜ回帰問題でよく使う二乗誤差（MSE）ではダメなのでしょうか？損失関数は「ただのスコア」ではなく、モデルが世界をどう解釈するかを規定する設計思想です。今日はその根拠を、情報理論・最尤推定・勾配の性質という三つの角度から徹底解剖します。

損失関数を選ぶことは、モデルが「何を最適化する生き物か」を決めることである。

2損失関数とは何か ─ 数学的な定義

損失関数 L は、モデルの予測 ŷ と真のラベル y のペアを受け取り、非負のスカラーを返す関数です。学習とはパラメータ θ を動かして L を最小化する過程そのものです。

経験損失の最小化：N サンプル平均の損失を最小にするパラメータ θ* を求める
f_θ(x) はモデル出力
損失は非負・予測が正解に一致するとき 0

3情報理論からクロスエントロピーを導く

エントロピーとは「驚き」の平均

ある事象が起きたときの「驚き（情報量）」は −log p で定義されます。確率が低いほど驚きは大きい、という直感と一致します。エントロピー確率分布が持つ不確実性・情報量の期待値 H(p) はその期待値です。

シャノンエントロピー：分布 p の不確実性を表すスカラー
自己情報量（驚き）の定義
エントロピーは驚きの期待値

クロスエントロピー ─ 「ズレた分布で符号化したコスト」

真の分布 p に対して、モデルの予測分布 q を使って符号化したとき、余計にかかる平均コストがクロスエントロピー真の分布 p とモデル分布 q の間の情報的距離を測る量です。

クロスエントロピー：p が真の分布、q がモデルの予測分布
KL ダイバージェンスとの関係
H(p) は定数なので最小化すると同値
分類での p は one-hot → 1 項だけ残る
⚙️ ひとことメモ
クロスエントロピーを最小化することは KL ダイバージェンスを最小化することと等価です。KL ダイバージェンスは非負（ギブスの不等式）なので、モデルが真の分布に近づくほど損失はゼロに近づきます。

4最尤推定 ─ 確率論からの別ルートで同じ答えに至る

情報理論とは独立した立場、統計の最尤推定観測データを最も「もっともらしく」説明するパラメータを求める手法（MLE）からもクロスエントロピーが導けます。

尤度最大化：全サンプルに対する条件付き確率の積を最大にするθを探す
対数を取って積→和に（単調増加なので等価）
最大化を最小化に反転（符号反転）
ソフトマックス出力 q = softmax(logits) を代入

こうして「尤度を最大化する」という統計的動機と「KL ダイバージェンスを最小化する」という情報理論的動機が、全く同じ損失関数 ─ クロスエントロピー ─ に収束します。二つの理論が独立に同じ答えを指す事実は、この損失関数の正当性を強く支持します。

2つ
情報理論・最尤推定という独立した理論がCEを導く
O(1)
one-hotラベル時の計算：正解クラスの対数確率1項のみ
KL≥0
ギブスの不等式が保証する損失の下界

5勾配の振る舞い ─ 二乗誤差が分類に向かない理由

ソフトマックス + MSE の「勾配消失問題」

分類モデルの最終層は ソフトマックスロジットを確率分布に変換する活性化関数 σ(z)_k = e^{z_k}/Σe^{z_j} を使います。MSE と組み合わせると逆伝播の勾配に σ′(z) = σ(1−σ) の項が掛かり、予測が確信的（0 か 1 に近い）なほど勾配が消えます。

MSE + sigmoid の勾配：予測が 0 か 1 に近いとき ŷ(1-ŷ)→0 で消える
CE + sigmoid の勾配（キャンセルして綺麗）
CE では σ′ の項が消え、誤差そのものが勾配になる

クロスエントロピーとソフトマックスを組み合わせると、勾配は ŷ − y（予測と正解の差）という直感的な形に整理されます。予測が大きく外れていれば勾配も大きく、学習が速い ─ これが実務上の最大のメリットです。

💡 R&Dでよく出る考え方
「損失関数 × 最終活性化関数」はセットで設計します。CE ↔ Softmax/Sigmoid、MSE ↔ 線形出力 ─ この組み合わせにはそれぞれ数学的必然性があります。論文でアーキテクチャを読む際は、この組み合わせが崩れていないか常にチェックしましょう。

6損失関数のバリエーション ─ 現代 AI 研究での展開

二値分類：バイナリクロスエントロピー（BCE）

BCE：正例・負例それぞれの対数確率を加算する二値版クロスエントロピー
K=2 の多値 CE と完全に等価

LLM のトークン予測：トークンレベル CE の平均

大規模言語モデル（GPT 系など）の学習損失は、各トークン位置でのクロスエントロピー損失モデルが次トークンを正しく予測できるかを測る損失 の系列平均です。これはパープレキシティの対数と直結します。

言語モデルの訓練損失：各ステップで次トークンを予測するCEの平均
パープレキシティとの関係

クラス不均衡への対処：Focal Loss

物体検出など、正例が極端に少ないタスクでは標準 CE だと負例の大量サンプルに勾配が支配されます。Focal Loss簡単なサンプルの損失を小さくして難しいサンプルに集中させるCEの拡張 はこれを解決します。

Focal Loss：γ>0 により確信度の高いサンプルの損失を指数的に減衰させる
γ=0 のとき通常の CE と一致

7実装のポイント ─ 数値安定性とよくある落とし穴

1def cross_entropy(logits, y_true):
2# Step1: log-sum-exp トリックで数値安定なソフトマックス
3c = max(logits) # オーバーフロー防止の定数
4log_sum_exp = log(sum(exp(logits - c))) + c
5# Step2: 正解クラスのログ確率だけ抜き出す
6log_prob = logits[y_true] - log_sum_exp
7# Step3: 符号反転して損失に
8return -log_prob
9
10# 注意：softmax → log → CE より log_softmax → nll_loss の方が安定
数値安定なクロスエントロピーの実装（NumPy 風擬似コード）
STEP 1

Softmax を先に計算してから log を取るのは危険。確率が 0 に丸められると log(0) = -∞ になる。log_softmax を一体で計算するライブラリ関数（PyTorch: F.log_softmax + F.nll_loss = F.cross_entropy）を使うこと。

STEP 2

ラベルスムージング（label smoothing）を加えると過信を防ぎ汎化性能が上がることがある。y を one-hot ではなく (1−ε)・one-hot + ε/K の分布として扱う手法で、Transformer 論文でも採用。

STEP 3

クラス不均衡があるときは weight 引数でクラスごとの損失重みを調整する。重みは「1 / クラス頻度」に比例させるのが基本戦略。

💡 R&Dでよく出る考え方
PyTorch の nn.CrossEntropyLoss は内部で log_softmax + NLL を合算しており、入力はロジット（softmax 前）を期待します。うっかり softmax 済みの出力を渡すと「softmax を二重適用」するバグになります。新しいモデルを実装したら必ずロジットか確率かを確認しましょう。

8まとめ ─ 設計思想を一本の糸でつなぐ

今日のポイントを整理します。クロスエントロピーは ①情報理論的に「予測分布と真の分布のズレ（KLダイバージェンス）」を最小化し、②確率論的に「尤度最大化（MLE）」と等価であり、③勾配計算では「ŷ − y」という美しい形に整理されます。この三つの一致が、分類問題でクロスエントロピーが選ばれ続ける数学的根拠です。

損失関数の選択は「実装の都合」ではなく「モデルに何を学ばせるかの哲学」です。明日以降、バックプロパゲーションと最適化アルゴリズムを学ぶとき、この損失の定義がすべての起点になることを思い出してください。
