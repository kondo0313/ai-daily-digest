1「線形だけ」では世界は表現できない

想像してください。あなたが猫と犬を見分けるとき、「耳の大きさ × 0.3 ＋ 鼻の形 × 0.7」のような単純な線形和だけで判断していますか？　実際は「耳が丸くて、かつ鼻がピンク色なら猫」という折れ曲がった複雑な境界が必要です。ニューラルネットワークも同じで、いくら層を深く重ねても、各層が線形変換だけなら全体は結局 1 枚の線形変換と等価になってしまいます。この「線形の呪い」を破るのが活性化関数各ニューロンの出力に加える非線形変換です。

活性化関数は「ただの数学的装飾」ではなく、深層学習が機能するための根本的な仕組みである。

今日は ReLU・GELU・SiLU という現代 AI で最も使われる 3 つの活性化関数を、数学的な定義から直感、そして LLM への応用まで一気に理解しましょう。DAY 6 で学んだ「層を重ねる意味」の答えがここに隠れています。

3ReLU ─ シンプルさが生んだ革命

定義と数式

ReLU は 2010 年代前半にDying ReLU 問題負の入力で勾配が 0 になり、ニューロンが永久に不活性化する問題を持ちながらも、それ以前の sigmoid・tanh を大幅に上回る学習速度を実現しました。理由は「正の領域で勾配が消えない」こと ─ 勾配消失問題深い層で誤差信号がほぼ 0 になり学習が止まる現象を大きく緩和したのです。

ReLU は「折れ線近似器」として機能します。ReLU を使った深いネットワークは、任意の連続関数を区分線形関数で近似していると見なせます。各ニューロンが「ここから先は傾き 1、手前は 0」という折れ目を学習し、組み合わせて複雑な形を作ります。

4GELU ─ 確率論的な「滑らかなReLU」

定義と導出の背景

GELU（Gaussian Error Linear Unit）は Hendrycks & Gimpel（2016）が提案した関数です。「入力をそのまま通すか、ゼロにするかを確率的に決める」という発想から生まれました。具体的には、入力 $x$ が標準正規分布に従う確率変数を超える確率 $\Phi(x)$ で、$x$ をそのまま通す確率が決まります。

GELU の確率的定義（X ~ N(0,1)）
Φ(x) の定義（標準正規分布の累積分布関数）
実装上の近似式（tanh版）

直感的なイメージ ─ 「確率ゲート」

ReLU が「$x > 0$ なら通す、$x < 0$ なら遮断する」というハードな二択だとすれば、GELU は「$x$ が大きいほど高確率で通し、$x$ が小さいほど低確率で通す」というソフトなゲートです。たとえ話で言うと、ReLU はガードマンが完全に扉を開け閉めする係、GELU は音量に応じてドアをじわじわ開けるオートドアのようなものです。

この滑らかさが勾配の安定性学習中に勾配が急激に変化しない性質をもたらし、BERT・GPT-2 以降の Transformer 系モデルで標準採用されるようになりました。

⚙️ ひとことメモ
GELU は x=0 付近でわずかに「負の値」を返すことがあります（x が少し負のとき Φ(x) が 0 より大きいため）。この微小な負値が表現力を高める一因とも言われています。ReLU と違い「完全な死（Dead Neuron）」が起きにくい性質もあります。

GELU との関係と使い分け

SiLU と GELU は形が非常に似ており、$\sigma(\sqrt{\pi/2}\,x) \approx \Phi(x)$ という近似関係があります。SiLU はシグモイド関数を自分自身のゲートとして使う「自己ゲーティング」の形をしており、計算コストが GELU より低いのが特徴です。SwiGLUSiLU を拡張したゲート付き線形ユニットで、LLaMA・PaLM 等で採用は SiLU をベースに構築されており、現代の大規模 LLM の FFN 層で主流になっています。

63 関数の比較 ─ どこが違うのか一目でわかる

ReLU

max(0,x)。最もシンプル。正領域で勾配=1、負領域で勾配=0。計算最速。Dying ReLU に注意。CNN・ResNet で広く使用。

GELU

x・Φ(x)。滑らかで確率論的解釈あり。x=0 付近でわずかに負値を返す。勾配が連続。BERT・GPT 系 Transformer の標準。

SiLU

x・σ(x)。自己ゲーティング構造。GELU より計算がシンプル。SwiGLU の基礎。LLaMA・Mistral 等の現代 LLM に採用。

7数学的性質の深掘り ─ 普遍近似定理との接続

なぜ非線形関数を挟んだ多層ネットワークが「任意の関数を近似できる」のでしょうか？　その理論的根拠が普遍近似定理1 つの隠れ層と非線形活性化があれば任意の連続関数を任意精度で近似できるという定理です。

普遍近似定理の形式（φ は非線形活性化関数）
近似誤差の上界（Cybenko 1989 の精神）

重要なのは「どんな非線形関数でも良いわけではない」という点です。シグモイド型や ReLU 型のように「非定数で有界（または部分線形）な関数」が条件を満たします。ReLU・GELU・SiLU はいずれもこの条件を満たし、かつ勾配消失を避けながら効率よく学習できる性質が加わっています。

8LLM の FFN 層における活性化関数 ─ SwiGLU という進化

現代の LLM（GPT-4、LLaMA、Gemini など）では、Transformer のFFN層各トークンに独立に適用される 2 層の全結合ネットワークが全パラメータの約 2/3 を占めます。この FFN の設計が性能を大きく左右します。

古典的な FFN（ReLU 版）
SwiGLU による置き換え（Noam Shazeer 2020）
⊙ はアダマール積（要素積）

SwiGLU は SiLU を使ったゲートと値の2系統の信号を要素積で合成します。これにより「どの情報を通すか」を学習可能なゲートが決める、より豊かな表現が可能です。実験的に ReLU 版 FFN より perplexity（言語モデルの評価指標）が大幅に改善することが報告されています。

10まとめ ─ 非線形性こそが「知性」の源泉

① 非線形の必要性

線形変換だけでは深い層が無意味。活性化関数が「深さを意味あるもの」にする。

② ReLU の革命

max(0,x) という単純さで勾配消失を緩和。CNN 時代の標準。折れ線近似器として機能。

③ GELU の滑らかさ

確率論的ゲートで滑らかな非線形性を実現。Transformer（BERT・GPT）の標準活性化。

④ SiLU と SwiGLU

自己ゲーティングで表現力を高め、LLaMA・PaLM などの大規模 LLM に採用される最先端 FFN 設計。

⑤ 次につながる視点

DAY 8 では「損失関数と最適化」へ進みます。活性化関数が変わると勾配の形が変わり、最適化にも影響する ── その接続に注目してください。

活性化関数を選ぶことは、ネットワークがどのように「世界の曲がりくねった構造」を学ぶかを決める、設計の核心である。
