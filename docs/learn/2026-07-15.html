1なぜ「一つの目」では足りないのか

「The bank by the river was steep.」という文を考えてみましょう。"bank" が「川岸」であることを理解するには、少なくとも「river」と「steep」の両方を同時に参照しなければなりません。さらに「was」は過去形として動詞の時制を解析する必要があり、「the」は名詞句の開始を示しています。一つのAttention機構入力系列の任意の位置同士の関連度を計算し、文脈に応じた表現を作る仕組みでは、これら複数の「視点」を同時に、かつ独立に捉えることが難しいのです。Multi-Head Attention は、まさにこの問題を解決するために設計されました。

一つの Attention ヘッドは「一つの視点」。Multi-Head Attention は、複数の独立した視点を並列に走らせ、最後に統合する設計思想だ。

3Multi-Head Attention の数学的定義

Multi-Head Attention の本質は、「同一の入力を h 個の異なる線形変換で射影し、それぞれ独立に Attention を計算した後、結果を連結して再射影する」という操作です。各ヘッド i に対して、専用の射影行列 W_i^Q, W_i^K, W_i^V が用意されます。

4直感：各ヘッドは何を「見て」いるのか

たとえ話を使いましょう。記者会見の動画を報道する際、一人の記者が「発言内容」「表情」「会場の雰囲気」をすべて同時に同等の注意で観察することは難しい。そこで複数の記者が分担します。一人目は言葉の内容に、二人目は感情表現に、三人目は前後の文脈の整合性に集中する——これが Multi-Head Attention の分業です。各ヘッドは異なる W_i^Q, W_i^K で「どこに注目するか」の視点が変わり、異なる表現空間データを埋め込む幾何学的な空間のことで類似度を計算します。

研究によって実際に確認されたヘッドの役割の例としては、「直前のトークンを強く参照するヘッド」「文の動詞と主語の対応を追うヘッド」「長距離依存（coreference）を捉えるヘッド」などがあります。全ヘッドが同一の動作をするわけではなく、学習によって自然に分業が生まれるのが美しい点です。

5設計の深掘り

① 計算量は Single-Head と同等になる理由

h 個のヘッドに分割すると「計算量が h 倍になるのでは？」と思うかもしれません。しかし各ヘッドの次元は d_model/h に圧縮されているため、トータルの行列積の計算コストはほぼ変わりません。

h ヘッドに分割しても総計算量はほぼ不変
d_k = d_model / h を代入
QK^⊤ の計算に加え射影コストも考慮すると

② 出力射影行列 W^O の役割

各ヘッドの出力を Concat した後に W^O で線形変換する理由は二つあります。一つ目は次元を d_model に戻すこと、二つ目は「各ヘッドの情報を混合・調整する」ことです。W^O は各ヘッドの寄与を重みづけしながら統合する「編集者」の役割を担います。この段階で初めて、異なる視点の情報が一つの埋め込みベクトルトークンの意味や文脈を数値のベクトルとして表現したものに統合されます。

③ Masked Multi-Head Attention（デコーダ側）

GPT などの自己回帰モデル過去のトークンのみから次のトークンを予測するモデルでは、未来のトークンを参照してはいけません。これを実現するのが Causal Mask です。softmax の前に「未来の位置」に -∞ を加算することで、それらの Attention 重みをゼロに強制します。

Causal Mask を加えた Masked Attention
Causal Mask M の定義（i < j の位置を -∞ でマスク）
softmax(-∞) = 0 なので未来への Attention は完全にゼロ

6現代 LLM への接続

GPT-4、LLaMA、Gemini など現代の大規模言語モデルはすべて Multi-Head Attention を中核に持ちます。ただし、スケールアップに伴っていくつかの改良が施されています。例えば Grouped-Query Attention (GQA) は、h 個のクエリヘッドに対してキー・バリューヘッドを g グループ（g < h）に削減する技術で、KV キャッシュのメモリを大幅に節約します。LLaMA 2 や Mistral で採用されています。また Multi-Query Attention (MQA) は GQA の極限（g = 1）で、全クエリが単一の K/V を共有します。

Grouped-Query Attention の定義（r はクエリヘッドとKVヘッドの比率）
g = h のとき（MHA）
g = 1 のとき（MQA）
💡 R&D でよく出る考え方
実務では「ヘッド数 h を増やす ≠ 精度が上がる」という事実が重要です。ヘッド数を増やしすぎると各ヘッドの次元 d_k が小さくなりすぎ、表現力が低下することがあります。LLM の設計では h と d_model のバランス調整が重要なハイパーパラメータチューニングの一つです。

7実装のヒント：PyTorch 風の擬似コード

1class MultiHeadAttention:
2def __init__(self, d_model, h):
3self.h = h
4self.d_k = d_model // h # 各ヘッドの次元
5self.W_Q = Linear(d_model, d_model) # 全ヘッド分まとめて射影
6self.W_K = Linear(d_model, d_model)
7self.W_V = Linear(d_model, d_model)
8self.W_O = Linear(d_model, d_model)
9
10def forward(self, Q, K, V, mask=None):
11B, n, _ = Q.shape # バッチ・系列長・次元
12# (B, n, d_model) → (B, h, n, d_k) に reshape
13Q = self.W_Q(Q).view(B, n, self.h, self.d_k).transpose(1,2)
14K = self.W_K(K).view(B, n, self.h, self.d_k).transpose(1,2)
15V = self.W_V(V).view(B, n, self.h, self.d_k).transpose(1,2)
16# Scaled Dot-Product Attention（各ヘッドに並列適用）
17scores = (Q @ K.transpose(-2,-1)) / sqrt(self.d_k)
18if mask is not None: scores = scores.masked_fill(mask, -1e9)
19attn = softmax(scores, dim=-1) # (B, h, n, n)
20out = attn @ V # (B, h, n, d_k)
21# ヘッドを連結して W_O で再射影
22out = out.transpose(1,2).contiguous().view(B, n, -1)
23return self.W_O(out) # (B, n, d_model)

⚙️ ひとことメモ
実装上のポイント：射影行列を「全ヘッドまとめて一つの大きな Linear 層」として実装すると、GPU のバッチ行列積（bmm）を最大限に活用でき、実際のスループットが大幅に向上します。ヘッドごとにループを回す実装は避けましょう。

8Multi-Head Attention の学習ダイナミクス

Multi-Head Attention は、表現学習データの有用な特徴を自動的に獲得する学習のことの観点からも重要です。各ヘッドの射影行列は、バックプロパゲーションを通じて「どの視点が損失関数を下げるか」を学習します。興味深いことに、学習済みモデルのヘッドを解析すると、一部のヘッドが冗長であることが知られており（Attention Head Pruning の研究）、全ヘッドが常に重要というわけではありません。これはスパース性と冗長性の許容がモデルの汎化能力に貢献していることを示唆しています。

STEP 1

入力 X を Q, K, V の三つに（または Encoder-Decoder では別々に）射影する

STEP 2

h 個の (W_i^Q, W_i^K, W_i^V) で異なる部分空間に変換し、各ヘッドで独立に Scaled Dot-Product Attention を計算

STEP 3

全ヘッドの出力を Concat し、W^O で d_model 次元に再射影して残差接続・Layer Norm へ送る

STEP 4

損失から逆伝播することで各ヘッドの射影行列が「専門化」し、異なる言語的パターンを担当するようになる

9まとめと次のステップ

Multi-Head Attention は「複数の独立した視点から入力を解析し、その知見を統合する」というシンプルかつ強力なアイデアです。数学的には、同一入力への複数の線形射影と Attention の並列適用、そして最終的な線形変換による統合として記述されます。実装コストは Single-Head と同等でありながら、表現の多様性と冗長性を獲得できるのが設計の妙です。
