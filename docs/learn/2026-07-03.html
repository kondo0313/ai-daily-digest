1なぜ「下る」のか ─ 学習の本質を問う

AI モデルは「パラメータ」と呼ばれる大量の数値を持っています。画像を正しく分類できるか、文章を自然に生成できるかは、すべてこのパラメータの値次第です。では、どうやって「正しい値」を見つけるのでしょうか？

ここで登場するのが損失関数モデルの予測と正解のずれを数値化した関数。小さいほどモデルが優秀です。学習とは「損失関数の値を最小にするパラメータを探す旅」に他なりません。そしてその旅の羅針盤が、今日学ぶ勾配降下法です。

「山の霧の中で最も低い谷を目指すとき、足元の傾きだけを頼りにする」── それが勾配降下法の直感。

2勾配降下法の数学的定義

パラメータベクトルを $\boldsymbol{\theta} \in \mathbb{R}^d$、損失関数を $L(\boldsymbol{\theta})$ とします。勾配降下法の更新則は以下の一行で表せます。

勾配降下法の基本更新則
η（学習率）は「一歩の大きさ」
∇L は損失が最も急上昇する方向
マイナスを付けることで「下り」方向へ進む

更新を繰り返すことで $L$ は（凸関数なら必ず、非凸でも局所的に）小さくなっていきます。$\eta$ が大きすぎると谷を飛び越えて発散し、小さすぎると収束が遅い ─ このトレードオフが実務で最も悩む点です。

η ≈ 0.01
よく使われる初期学習率の目安
d ≈ 10⁹
現代の大規模 LLM のパラメータ数オーダー
O(d)

33つの「バリエーション」を理解する

① バッチ勾配降下法 (Batch GD)

訓練データ全 $N$ サンプルを使って勾配を計算します。

全サンプル平均による真の勾配推定
各サンプルの損失 ℓ の勾配を平均

利点は安定した更新ですが、$N$ が巨大だと1ステップの計算が重すぎて実用的ではありません。

② 確率的勾配降下法 (SGD)

毎ステップランダムに1サンプルを選んで勾配を近似します。SGDStochastic Gradient Descent の略。ノイズが多いが高速で汎化性能が高い傾向の更新則はシンプルです。

SGD の更新則（添字 i_t はランダムに選ばれたサンプル）
勾配の不偏推定量になっている

勾配にノイズが乗るため「鞍点や浅い極小を脱出しやすい」という嬉しい副作用があります。実務ではミニバッチ（32〜256サンプル）を使うミニバッチ SGD が主流です。

③ モメンタム付き SGD

勾配の「過去の流れ」を蓄積し、滑らかに加速するのがモメンタム法です。ボールが坂を転がり慣性で加速するイメージです。

モメンタム付き SGD の更新則
β（モメンタム係数）は慣性の強さ。典型値は 0.9
v を展開すると過去の勾配の指数和になる

4現代の主役 ─ Adam アルゴリズムの解剖

2015年に Kingma & Ba が発表した AdamAdaptive Moment Estimation の略。勾配の1次・2次モーメントを使って学習率を自動調整する最適化手法 は、今も深層学習の事実上の標準です。

Adam の天才的アイデアは「各パラメータに異なる学習率を自動で与える」こと。

Adam は1次モーメント（平均）と2次モーメント（分散）の両方を追跡します。

Adam の1次・2次モーメント推定（g_t は時刻 t の勾配）
m は勾配の指数移動平均（方向の安定化）
v は勾配の二乗の指数移動平均（スケールの把握）

初期ステップでは $\boldsymbol{m}, \boldsymbol{v}$ がゼロ初期化されているためバイアス補正が必要です。

バイアス補正済みモーメント推定
t が小さいときほど補正が大きく効く
β^t → 0 になるにつれ補正不要になる

最終的な更新則は次のとおりです。

Adam の最終更新式（除算・√はすべて要素ごと）
分母 √v̂ + ε が実効学習率を自動スケーリング
ε はゼロ除算防止の微小定数
💡 R&Dでよく出る考え方
Adam の√v̂ は「これまでの勾配の大きさ」を表します。ある次元でずっと大きな勾配が来ていれば学習率が下がり、まれにしか更新されない次元（埋め込みの稀な単語など）では学習率が上がります。この性質がスパースな特徴量を持つ NLP タスクで特に有効です。

5擬似コードで理解するアルゴリズムの流れ

1# 初期化
2m = zeros_like(θ) # 1次モーメント
3v = zeros_like(θ) # 2次モーメント
4β1, β2, η, ε = 0.9, 0.999, 1e-3, 1e-8
5
6for t in range(1, T+1):
7g = compute_gradient(L, θ) # 勾配を計算
8m = β1 * m + (1 - β1) * g # 1次モーメント更新
9v = β2 * v + (1 - β2) * g**2 # 2次モーメント更新
10m_hat = m / (1 - β1**t) # バイアス補正
11v_hat = v / (1 - β2**t) # バイアス補正
12θ = θ - η * m_hat / (sqrt(v_hat) + ε) # パラメータ更新

6学習率スケジューリング ─ 旅の途中でペースを変える

固定学習率では収束直前に「振動」が起きやすいため、学習率スケジューラ学習の進行に合わせて η を動的に変化させる仕組みが広く使われます。代表的な戦略を整理します。

STEP 1

ウォームアップ: 最初の数千ステップで η を 0 から徐々に上げる。学習初期の不安定さを防ぐ。

STEP 2

コサインアニーリング: η を半周期のコサイン曲線でなめらかに減衰させる。η_t = η_min + (η_max - η_min)/2 × (1 + cos(πt/T)) の形。

STEP 3

線形減衰 / ステップ減衰: 一定ステップごとに η に係数（例: 0.1）を掛ける。実装がシンプルで直感的。

STEP 4

ポリシーを組み合わせる: Transformer 系モデルでは「ウォームアップ + 逆数乗根減衰」が Attention is All You Need 論文以来の慣習。

Transformer オリジナル論文のスケジューリング式
t < t_warmup のとき: η は線形増加フェーズ
t > t_warmup のとき: η は t^{-0.5} で減衰

7現代 LLM の最適化 ─ R&D 現場のリアル

GPT・LLaMA・Gemini などの大規模言語モデルの学習では、Adam の派生系が使われています。代表的なものに AdamW があります。これは Adam に重み減衰L2 正則化に相当する項を加え、過学習を防ぐテクニックを正しく組み込んだもので、更新式に次の項が加わります。

AdamW の更新式（λ が重み減衰係数）
Adam の勾配項に重み減衰 λθ を加算
L2 正則化と異なり、適応的スケーリングの影響を受けない
⚙️ ひとことメモ
最近の研究では Muon（モーメント行列を直交化する最適化手法）や SOAP、Lion など Adam を超えるアルゴリズムが次々と提案されています。しかし実務上は「AdamW + 適切なスケジューラ + 良いバッチサイズ」の組み合わせが依然として最強に近く、R&D では「新手法を試す前に基本設定を丁寧にチューニングする」姿勢が重要です。

8可視化で掴む ─ 損失曲面と最適化の振る舞い

損失曲面は複雑な高次元空間ですが、2次元に投影すると「丘あり、谷あり、鞍あり」の地形が浮かびます。この地形で各手法がどう動くか整理しましょう。

谷間（細長い谷）

バニラ SGD はジグザグに進む。モメンタムを加えると谷に沿って加速し、より速く収束。

鞍点（偏微分がゼロだが極小でない点）

高次元では極めて多い。SGD のノイズや Adam の適応的学習率が自然に脱出を助ける。

局所極小

深層ネットワークでは多く存在するが、多くは「フラットな谷底」でバリデーション性能が良好。真に悪い局所極小に陥るケースは実は少ない。

プラトー（勾配がほぼゼロ）

モメンタムや Adam の過去の速度蓄積により通過しやすくなる。

💡 R&Dでよく出る考え方
損失曲面の「フラットさ（Sharpness）」と汎化性能には深い関係があります。SAM（Sharpness-Aware Minimization）は「フラットな極小を意図的に探す」アルゴリズムで、ImageNet・NLP 双方で汎化向上が報告されています。最適化の最前線は「どこに収束するか」だけでなく「どんな形の谷に落ちるか」にシフトしています。

9まとめ ─ 明日へつながる問い

今日学んだことを整理します。勾配降下法は「損失関数の勾配方向に逆らって進む」というシンプルな原理から出発し、SGD → モメンタム → Adam → AdamW と進化してきました。その本質はすべて「どの方向に・どれだけ進むか」の問いへの答えです。

基礎

勾配降下法: θ ← θ - η∇L。勾配の反対方向に学習率分だけ進む。

高速化

SGD + ミニバッチ: 全データでなくランダムサンプルで勾配を近似。

安定化

モメンタム: 過去の勾配を慣性として活用。鞍点・プラトーを越えやすい。

自動調整

Adam / AdamW: 各パラメータに適応的な学習率。LLM 訓練の事実上の標準。

収束改善

学習率スケジューラ: ウォームアップ + 減衰で安定した収束を実現。

最適化は AI の「エンジン」。数式の背後にある物理的直感を掴めた研究者が、モデルの挙動を本当に制御できる。
