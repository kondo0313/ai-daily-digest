Bifocal Diffusion Language Models: Asymmetric Bidirectional Context for Parallel Generation

1「速いか、賢いか」──LLM生成の二律背反を超える

ChatGPT のような大規模言語モデルは、文章を「左から右へ、1トークンずつ」生成します。これを自己回帰（AR）生成前のトークンを見ながら次を1個ずつ予測する生成方式。正確だが遅いといいます。速度を上げるには並列化が必要ですが、生成物の品質と速度を同時に高めることが非常に難しく、研究者たちは長年この壁に悩んできました。

そこに登場したのが 拡散言語モデル（dLLM: discrete diffusion language model） です。画像生成で有名な拡散モデルノイズを徐々に除去して高品質なデータを生成するモデル。Stable Diffusion が代表例を言語に応用したもので、マスクされた複数のトークンを「一度に」復元できます。理論上は大幅なスピードアップが期待できるのですが……実用化には厄介な設計上の問題が立ちはだかっていました。

双方向を選べば賢いが遅い。因果的を選べば速いが頭が悪い。この二択こそが、拡散言語モデルの普及を阻んでいた根本的なジレンマだった。

2ジレンマの正体──アテンションとKVキャッシュの相性問題

dLLM でトークンを並列復元するとき、どの範囲の文脈を「見て」いいか、という設計選択があります。

選択肢 A

双方向アテンション：左右のすべての文脈を参照できるので生成品質が高い。しかしKVキャッシュTransformerが過去の文脈の Key・Value を保存して再計算を省くメモリ上の仕組みが使えず、バッチ処理でスループットが大幅に落ちる。

選択肢 B

因果的アテンション：左側しか見ないため KV キャッシュは使えるが、右側の文脈が一切参照できず生成品質が著しく低下する。dLLM の強みである「並列復元」の恩恵が消えてしまう。

ごく簡単に言えば「品質か速度か」の二択です。そしてこれは設計上の些細なトレードオフではなく、アーキテクチャの根幹に関わる構造的な問題でした。

3Bifocal dLLM の発想──「左のアテンション」と「右のMamba」を組み合わせる

この論文が提案するのが Bifocal dLLM という新しいパラダイムです。名前の「Bifocal（バイフォーカル）」は老眼鏡の「遠近両用レンズ」から来ています。遠くは別の焦点で見て、近くも見える──その発想を言語モデルのアーキテクチャに応用しました。

具体的な実装が R2LM（Right-to-Left Mamba） です。二つのコンポーネントで構成されます。① 標準の因果的アテンション：左側（過去）の文脈を精密に参照し、KV キャッシュも完全に利用できる。② 逆方向 Mamba SSM状態空間モデル(SSM)の一種。RNNに似た再帰構造で長系列を効率処理するニューラルモジュールサイドカー：右側（未来）の文脈を「圧縮した形で」提供する軽量な追加モジュール。この二つを組み合わせることで、右側の文脈を完全には見えないが「圧縮した要約」として参照しつつ、左側はキャッシュ付きの因果的アテンションで高精度に処理します。

# ① マスクされたトークン列を受け取る
1x = mask_tokens(input_ids, mask_ratio)①
# ② 逆方向Mambaで右側の文脈を圧縮して要約
2right_ctx = reverse_mamba(x)②
# ③ 因果的アテンション（KVキャッシュ付き）で左側を精密参照
3left_ctx = causal_attention(x, kv_cache=True)③
# ④ 両者を統合してマスクトークンを並列復元
4hidden = left_ctx + right_ctx
5tokens = parallel_decode(hidden)④

① マスクトークンの並列復元とは

dLLM は画像のインペインティング画像の一部を塗りつぶして埋め直す処理。拡散モデルが得意とするに似た操作を言語で行います。文章中の複数の位置を同時に「マスク（空欄）」にして、それらを同時に予測・復元します。AR 生成が「1問ずつ解く」なら、dLLM は「全問同時に解く」イメージです。

② 逆方向 Mamba SSM サイドカーの役割

Mamba はもともと「左から右」に文脈を圧縮していく状態空間モデルです。これを 逆方向（右から左）に走らせると、「右側の未来文脈の圧縮要約」が得られます。この要約は完全な双方向アテンションには及ばないものの、「何もない状態」よりずっとリッチな右側情報を提供します。そして Mamba は RNN ライクな構造なので、KV キャッシュを壊しません。サイドカーなので本体の因果的アテンションに追加するだけで良く、実装が比較的シンプルなのも利点です。

③ 因果的アテンション + KV キャッシュで高速化

本体の Transformer ブロックは通常の因果的アテンションを使います。KV キャッシュが使えるので、複数ユーザーが同時リクエストするバッチサービングでも推論が高速です。dLLM の「並列デコーディング（複数トークンを一度に復元）」と KV キャッシュを組み合わせることで、AR 生成を大きく上回るスループットを達成しています。

5なぜこれが重要なのか──dLLM 実用化への扉

R2LM は「並列生成の品質」と「KV キャッシュによる高速推論」という、互いに相容れないと思われていた要件を同時に達成した。これは拡散言語モデルが実用 API として登場する未来への重要な一歩だ。

現在の大規模サービス（ChatGPT、Claude など）はすべて AR 生成です。dLLM が実用的なスループットを持てれば、より安いコスト・より短いレイテンシで同等品質の回答を提供できる可能性があります。この研究はその実現可能性を具体的な数値で示した点で意義深いです。

6意義と限界──論文が認める弱点も正直に

限界 1

サイドカーの Mamba は右側文脈を「圧縮」するため、完全な双方向アテンションには情報量で劣る。特に非常に長い依存関係があるタスクでは差が出る可能性がある。

限界 2

実験は Qwen3-1.7B という比較的小さなモデルで行われており、7B・70B といった大規模モデルへのスケールアップ検証はこれから。

限界 3

継続事前学習（60B トークン）という設定であり、ゼロからの大規模学習での性能はまだ未知数。

限界 4

スループット比較の条件（バッチサイズ、シーケンス長、ハードウェア）によって倍率が大きく変わる（2.4〜12.9× の幅はそれを反映）。
